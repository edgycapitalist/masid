## Overview

The URL shortening library is designed to provide a lightweight, thread-safe, and efficient solution for shortening URLs using a hash-based approach. The library will support reverse lookup from short URL to original URL, handle collision detection, and include input validation for valid URL formats.

## System Components

1. **URL Shortener**: Responsible for generating a shortened URL from a given long URL.
2. **URL Store**: A data structure that stores the mapping between shortened URLs and their corresponding original URLs.
3. **Collision Detector**: A mechanism to detect and handle collisions when two different long URLs generate the same shortened URL.

## System Design

The library will utilize a hash-based approach to generate shortened URLs. The hash function will take the long URL as input and produce a fixed-size hash value, which will be used as the shortened URL.

### Hash Function

The hash function will be implemented using the SHA-256 algorithm, which is a widely used and cryptographically secure hash function. The hash value will be truncated to a fixed size (e.g., 6 characters) to generate the shortened URL.

### URL Store

The URL store will be implemented using a thread-safe dictionary (e.g., `concurrent.futures.ThreadPoolExecutor` with a `dict` backend). The dictionary will store the mapping between shortened URLs and their corresponding original URLs.

### Collision Detector

The collision detector will be implemented using a simple counter-based approach. When a collision is detected, the counter will be incremented, and the hash value will be re-generated using the incremented counter as a salt value.

## Input Validation

The library will include input validation to ensure that the input URL is in a valid format. The validation will be performed using the `urllib.parse` module.

## Thread Safety

The library will be designed to be thread-safe for concurrent usage. The `concurrent.futures` module will be used to implement thread-safe data structures and synchronization primitives.

import hashlib
import threading
from urllib.parse import urlparse
from concurrent.futures import ThreadPoolExecutor

class URLShortener:
    def __init__(self):
        self.url_store = {}
        self.lock = threading.Lock()
        self.counter = 0

    def shorten_url(self, long_url):
        with self.lock:
            if not self.is_valid_url(long_url):
                raise ValueError("Invalid URL format")

            hash_value = self.generate_hash(long_url)
            shortened_url = hash_value[:6]

            if shortened_url in self.url_store:
                self.handle_collision(shortened_url, long_url)
            else:
                self.url_store[shortened_url] = long_url

            return shortened_url

    def get_original_url(self, shortened_url):
        with self.lock:
            return self.url_store.get(shortened_url)

    def is_valid_url(self, url):
        try:
            result = urlparse(url)
            return all([result.scheme, result.netloc])
        except ValueError:
            return False

    def generate_hash(self, long_url):
        return hashlib.sha256(long_url.encode()).hexdigest()

    def handle_collision(self, shortened_url, long_url):
        self.counter += 1
        new_hash_value = self.generate_hash(long_url + str(self.counter))
        new_shortened_url = new_hash_value[:6]
        self.url_store[new_shortened_url] = long_url
        return new_shortened_url
