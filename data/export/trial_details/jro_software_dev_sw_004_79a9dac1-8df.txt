## Overview

The in-memory task queue is a critical component of our project, responsible for managing tasks with varying priorities, tracking their status, and ensuring reliable execution. This document outlines the high-level design, module decomposition, interfaces, data flow, and technology choices for the task queue.

## Module Decomposition

The task queue will consist of the following modules:

1. **Task**: Represents a single task with its priority, status, and other relevant attributes.
2. **TaskQueue**: Manages the task queue, including adding tasks, retrieving tasks, and tracking task status.
3. **Worker**: Responsible for consuming tasks from the task queue and executing them.
4. **RetryPolicy**: Defines the retry strategy for failed tasks, including the maximum number of retries.
5. **ShutdownManager**: Handles graceful shutdown of the task queue, ensuring that all tasks are completed or cancelled.

## Interfaces

The following interfaces will be used to interact with the task queue:

1. **TaskQueueInterface**: Defines methods for adding tasks, retrieving tasks, and tracking task status.
2. **WorkerInterface**: Defines methods for consuming tasks from the task queue and executing them.
3. **RetryPolicyInterface**: Defines methods for determining the retry strategy for failed tasks.

## Data Flow

The data flow between modules will be as follows:

1. **Task** objects are created and added to the **TaskQueue**.
2. **Worker** objects consume tasks from the **TaskQueue** and execute them.
3. The **TaskQueue** tracks the status of each **Task** object, updating its status as it is executed.
4. If a **Task** fails, the **RetryPolicy** is consulted to determine if the task should be retried.
5. If a task is retried, the **TaskQueue** will re-add the task to the queue with a new attempt count.

## Technology Choices

The task queue will be implemented using Python, leveraging the following libraries:

1. **queue**: For implementing the in-memory task queue.
2. **threading**: For managing worker threads that consume tasks from the queue.
3. **enum**: For defining task status and other enumerations.

```python
import queue
import threading
from enum import Enum
import time

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, priority, func, args=None, kwargs=None):
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.status = TaskStatus.PENDING
        self.attempts = 0

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.tasks = {}
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.queue.put((task.priority, task))
            self.tasks[task] = task.status

    def get_task(self):
        with self.lock:
            if not self.queue.empty():
                _, task = self.queue.get()
                return task
            return None

    def update_task_status(self, task, status):
        with self.lock:
            self.tasks[task] = status

class Worker(threading.Thread):
    def __init__(self, task_queue, retry_policy):
        super().__init__()
        self.task_queue = task_queue
        self.retry_policy = retry_policy
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(0.1)
                continue
            try:
                task.status = TaskStatus.RUNNING
                self.task_queue.update_task_status(task, TaskStatus.RUNNING)
                task.func(*task.args, **task.kwargs)
                task.status = TaskStatus.COMPLETED
                self.task_queue.update_task_status(task, TaskStatus.COMPLETED)
            except Exception as e:
                task.attempts += 1
                task.status = TaskStatus.FAILED
                self.task_queue.update_task_status(task, TaskStatus.FAILED)
                if self.retry_policy.should_retry(task):
                    self.task_queue.add_task(task)
                else:
                    print(f"Task failed after {task.attempts} attempts")

class RetryPolicy:
    def __init__(self, max_retries):
        self.max_retries = max_retries

    def should_retry(self, task):
        return task.attempts < self.max_retries

class ShutdownManager:
    def __init__(self, task_queue):
        self.task_queue = task_queue

    def shutdown(self):
        # Cancel all tasks in the queue
        while not self.task_queue.queue.empty():
            _, task = self.task_queue.queue.get()
            task.status = TaskStatus.COMPLETED
            self.task_queue.update_task_status(task, TaskStatus.COMPLETED)

def example_task():
    print("Task executed")

def main():
    task_queue = TaskQueue()
    retry_policy = RetryPolicy(3)
    worker = Worker(task_queue, retry_policy)
    worker.start()

    task_queue.add_task(Task(1, example_task))
    task_queue.add_task(Task(2, example_task))
    task_queue.add_task(Task(3, example_task))

    time.sleep(5)
    shutdown_manager = ShutdownManager(task_queue)
    shutdown_manager.shutdown()
    worker.running = False
    worker.join()

if __name__ == '__main__':
    main()
```
