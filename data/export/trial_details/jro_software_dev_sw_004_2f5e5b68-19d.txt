**Architecture Document: In-Memory Task Queue**

## Introduction

The in-memory task queue is designed to support adding tasks with priorities, allow worker threads to consume tasks, provide task status tracking, support task retry with configurable max retries, and include graceful shutdown. This document outlines the high-level structure, module decomposition, interfaces, data flow, and technology choices for the task queue.

## Module Decomposition

The task queue will consist of the following modules:

1. **Task**: Represents a single task with a unique ID, priority, and retry count.
2. **TaskQueue**: Manages the task queue, including adding tasks, removing tasks, and tracking task status.
3. **Worker**: Consumes tasks from the task queue and executes them.
4. **StatusTracker**: Tracks the status of each task (pending, running, completed, failed).
5. **RetryManager**: Manages task retry with configurable max retries.
6. **ShutdownManager**: Handles graceful shutdown of the task queue.

## Interfaces

The following interfaces will be used to interact with the task queue:

1. **TaskQueueInterface**: Defines methods for adding tasks, removing tasks, and tracking task status.
2. **WorkerInterface**: Defines methods for consuming tasks and executing them.
3. **StatusTrackerInterface**: Defines methods for tracking task status.
4. **RetryManagerInterface**: Defines methods for managing task retry.
5. **ShutdownManagerInterface**: Defines methods for handling graceful shutdown.

## Data Flow

The data flow for the task queue is as follows:

1. Tasks are added to the task queue through the **TaskQueueInterface**.
2. The **TaskQueue** module manages the task queue and assigns tasks to worker threads.
3. Worker threads consume tasks from the task queue through the **WorkerInterface**.
4. The **StatusTracker** module tracks the status of each task.
5. The **RetryManager** module manages task retry with configurable max retries.
6. The **ShutdownManager** module handles graceful shutdown of the task queue.

## Technology Choices

The task queue will be implemented in Python using the following technologies:

1. **Queue**: A thread-safe queue implementation, such as `queue.Queue`, will be used to manage the task queue.
2. **Thread**: The `threading` module will be used to implement worker threads.
3. **Lock**: The `threading.Lock` class will be used to synchronize access to shared resources.

```python
import threading
import time

class Task:

        self.status_tracker.track_status(task)

    def remove_task(self, task_id):

        self.status_tracker.update_status(task, "removed")

class Worker:
    def __init__(self, task_queue):
        self.task_queue = task_queue
        self.thread = threading.Thread(target=self.consume_tasks)

    def consume_tasks(self):
        while True:
            task = self.task_queue.queue.get()
            if task is None:
                break
            try:
                task.status = "running"
                self.task_queue.status_tracker.update_status(task, "running")
                time.sleep(1)  # simulate task execution
                task.status = "completed"
                self.task_queue.status_tracker.update_status(task, "completed")
            except Exception as e:
                task.retry_count -= 1
                if task.retry_count <= 0:
                    task.status = "failed"
                    self.task_queue.status_tracker.update_status(task, "failed")
                else:
                    self.task_queue.retry_manager.retry_task(task)

class StatusTracker:
    def __init__(self):
        self.status_map = {}

    def track_status(self, task):
        self.status_map[task.id] = task.status

class RetryManager:
```

The in-memory task queue is designed to support adding tasks with priorities, allow worker threads to consume tasks, provide task status tracking, support task retry with configurable max retries, and include graceful shutdown. This document outlines the high-level structure, module decomposition, interfaces, data flow, and technology choices for the task queue.

## Module Decomposition

The task queue will consist of the following modules:

1. **Task**: Represents a single task with a unique ID, priority, and retry count.
2. **TaskQueue**: Manages the task queue, including adding tasks, removing tasks, and tracking task status.
3. **Worker**: Consumes tasks from the task queue and executes them.
4. **StatusTracker**: Tracks the status of each task (pending, running, completed, failed).
5. **RetryManager**: Manages task retry with configurable max retries.
6. **ShutdownManager**: Handles graceful shutdown of the task queue.

## Interfaces

The following interfaces will be used to interact with the task queue:

1. **TaskQueueInterface**: Defines methods for adding tasks, removing tasks, and tracking task status.
2. **WorkerInterface**: Defines methods for consuming tasks and executing them.
3. **StatusTrackerInterface**: Defines methods for tracking task status.
4. **RetryManagerInterface**: Defines methods for managing task retry.
5. **ShutdownManagerInterface**: Defines methods for handling graceful shutdown.

## Data Flow

The data flow for the task queue is as follows:

1. Tasks are added to the task queue through the **TaskQueueInterface**.
2. The **TaskQueue** module manages the task queue and assigns tasks to worker threads.
3. Worker threads consume tasks from the task queue through the **WorkerInterface**.
4. The **StatusTracker** module tracks the status of each task.
5. The **RetryManager** module manages task retry with configurable max retries.
6. The **ShutdownManager** module handles graceful shutdown of the task queue.

## Technology Choices

The task queue will be implemented in Python using the following technologies:

1. **Queue**: A thread-safe queue implementation, such as `queue.Queue`, will be used to manage the task queue.
2. **Thread**: The `threading` module will be used to implement worker threads.
3. **Lock**: The `threading.Lock` class will be used to synchronize access to shared resources.

```python
import queue
import threading
import time
import logging
import os

# Set up logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class Task:
    def __init__(self, id, priority, retry_count):
        self.id = id
        self.priority = priority
        self.retry_count = retry_count
        self.status = "pending"

class TaskQueue:
    def __init__(self, max_retries=3):
        self.queue = queue.Queue()
        self.status_tracker = StatusTracker()
        self.retry_manager = RetryManager(max_retries)
        self.shutdown_manager = ShutdownManager()
        self.max_retries = max_retries

    def add_task(self, task):
        self.queue.put(task)
        self.status_tracker.track_status(task)

    def remove_task(self, task_id):
        task = self.queue.get()
        if task.id == task_id:
            self.status_tracker.update_status(task, "removed")
        else:
            self.queue.put(task)

    def start_workers(self, num_workers):
        self.workers = []
        for _ in range(num_workers):
            worker = Worker(self)
            worker.thread.start()
            self.workers.append(worker)

    def shutdown(self):
        self.shutdown_manager.shutdown_queue(self.queue)
        for worker in self.workers:
            worker.thread.join()

class Worker:
    def __init__(self, task_queue):
        self.task_queue = task_queue
        self.thread = threading.Thread(target=self.consume_tasks)

    def consume_tasks(self):
        while True:
            task = self.task_queue.queue.get()
            if task is None:
                break
            try:
                task.status = "running"
                self.task_queue.status_tracker.update_status(task, "running")
                time.sleep(1)  # simulate task execution
                task.status = "completed"
                self.task_queue.status_tracker.update_status(task, "completed")
            except Exception as e:
                logger.error(f"Error executing task {task.id}: {e}")
                task.retry_count -= 1
                if task.retry_count <= 0:
                    task.status = "failed"
                    self.task_queue.status_tracker.update_status(task, "failed")
                else:
                    self.task_queue.retry_manager.retry_task(task, self.task_queue)

class StatusTracker:
    def __init__(self):
        self.status_map = {}

    def track_status(self, task):
        self.status_map[task.id] = task.status

    def update_status(self, task, status):
        self.status_map[task.id] = status

class RetryManager:
    def __init__(self, max_retries):
        self.max_retries = max_retries

    def retry_task(self, task, task_queue):
        if task.retry_count <= 0:
            return
        task.status = "pending"
        task_queue.add_task(task)

class ShutdownManager:
    def __init__(self):
        self.shutdown = False

    def shutdown_queue(self, queue):
        self.shutdown = True
        while not queue.empty():
            task = queue.get()
            task.status = "removed"
            logger.info(f"Task {task.id} removed")

# Example usage:
task_queue = TaskQueue(max_retries=int(os.environ.get("MAX_RETRIES", 3)))
task_queue.start_workers(2)

task1 = Task(1, 1, 3)
task2 = Task(2, 2, 3)

task_queue.add_task(task1)
task_queue.add_task(task2)

time.sleep(5)

task_queue.shutdown()

print(task_queue.status_tracker.status_map)
```
