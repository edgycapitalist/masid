**Architecture Document: In-Memory Task Queue**

## Introduction

The in-memory task queue is designed to manage tasks with priorities, track task status, and provide task retry with configurable max retries. The system consists of the following components:

*   **Task**: Represents a single task with a unique ID, priority, and a callable function to execute.
*   **Task Queue**: A data structure that stores tasks with their corresponding priorities.
*   **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
*   **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).

## System Components

### Task

*   **ID**: Unique identifier for the task.
*   **Priority**: Priority of the task (higher value means higher priority).
*   **Callable Function**: The function to be executed when the task is consumed by a worker thread.
*   **Max Retries**: Maximum number of times the task can be retried if it fails.

### Task Queue

*   **Data Structure**: A priority queue (implemented using `heapq` in Python) to store tasks based on their priorities.
*   **Methods**:
    *   `add_task(task)`: Adds a task to the queue.
    *   `get_task()`: Retrieves the task with the highest priority from the queue.

### Worker Thread

*   **Thread**: A separate thread that consumes tasks from the task queue.
*   **Methods**:
    *   `start()`: Starts the worker thread.
    *   `stop()`: Stops the worker thread.

### Task Status Tracker

*   **Data Structure**: A dictionary to store the status of each task.
*   **Methods**:
    *   `update_status(task_id, status)`: Updates the status of a task.

## System Interactions

1.  **Adding a Task**:
    *   The client adds a task to the task queue using the `add_task` method.
    *   The task is stored in the priority queue based on its priority.
2.  **Consuming a Task**:
    *   A worker thread retrieves the task with the highest priority from the task queue using the `get_task` method.
    *   The worker thread executes the task and updates its status using the `update_status` method.
3.  **Task Retry**:
    *   If a task fails, the worker thread checks if the task has reached its maximum number of retries.
    *   If not, the task is added back to the task queue with its priority updated.
4.  **Graceful Shutdown**:
    *   When the system is shut down, all worker threads are stopped using the `stop` method.
    *   Any pending tasks are stored in the task queue for future execution.

import heapq
import threading
from enum import Enum
from typing import Callable

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id: str, priority: int, func: Callable, max_retries: int = 3):
        self.id = id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()
        self.status_tracker = {}

    def add_task(self, task: Task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))
            self.status_tracker[task.id] = task.status

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

    def update_status(self, task_id: str, status: TaskStatus):
        with self.lock:
            self.status_tracker[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = False

    def start(self):
        self.running = True
        super().start()

    def stop(self):
        self.running = False

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                task.status = TaskStatus.RUNNING
                self.task_queue.update_status(task.id, task.status)
                try:
                    task.func()
                    task.status = TaskStatus.COMPLETED
                except Exception as e:
                    task.status = TaskStatus.FAILED
                    task.retries += 1
                    if task.retries < task.max_retries:
                        self.task_queue.add_task(task)
                self.task_queue.update_status(task.id, task.status)

# Example usage
def example_task():
    print("Task executed successfully")

task_queue = TaskQueue()
worker_thread = WorkerThread(task_queue)

task = Task("example_task", 1, example_task)
task_queue.add_task(task)

worker_thread.start()
# Let the worker thread run for 5 seconds
import time
time.sleep(5)
worker_thread.stop()

## Introduction

*   **Task Queue**: A data structure that stores tasks based on their priorities.
*   **Worker**: Consumes tasks from the task queue, executes them, and updates their status.

### Worker

    *   `start()`: Starts the worker.
    *   `stop()`: Stops the worker.
