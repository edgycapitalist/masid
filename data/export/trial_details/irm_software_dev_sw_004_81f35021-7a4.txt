**Architecture Document: In-Memory Task Queue**

**Overview**
----------

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status and providing retry mechanisms. The system will ensure a graceful shutdown when required.

**Components**
------------

1. **Task**: Represents a unit of work with a priority and a callback function.
2. **Task Queue**: Manages tasks in memory, providing methods for adding, consuming, and tracking tasks.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of tasks (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable maximum number of attempts.

**Module Decomposition**
----------------------

The system will be decomposed into the following modules:

1. `task.py`: Defines the `Task` class.
2. `task_queue.py`: Implements the `TaskQueue` class.
3. `worker.py`: Defines the `WorkerThread` class.
4. `status_tracker.py`: Implements the `TaskStatusTracker` class.
5. `retry_mechanism.py`: Defines the `RetryMechanism` class.

**Interfaces**
-------------

1. `TaskQueue`:
	* `add_task(task: Task)`: Adds a task to the queue.
	* `get_task()`: Retrieves a task from the queue.
	* `update_task_status(task_id: int, status: str)`: Updates the status of a task.
2. `WorkerThread`:
	* `consume_task()`: Consumes a task from the task queue.
	* `execute_task(task: Task)`: Executes a task.
3. `TaskStatusTracker`:
	* `get_task_status(task_id: int)`: Retrieves the status of a task.
	* `update_task_status(task_id: int, status: str)`: Updates the status of a task.
4. `RetryMechanism`:
	* `retry_task(task: Task)`: Retries a task if it has failed.

**Data Flow**
------------

1. A task is added to the task queue using the `add_task` method.
2. A worker thread consumes a task from the task queue using the `get_task` method.
3. The worker thread executes the task using the `execute_task` method.
4. The task status is updated using the `update_task_status` method.
5. If a task fails, the retry mechanism is triggered using the `retry_task` method.

import heapq
import threading
from dataclasses import dataclass
from typing import Callable

@dataclass
class Task:
    id: int
    priority: int
    callback: Callable
    max_retries: int = 3

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            return heapq.heappop(self.queue)[1]

    def update_task_status(self, task_id: int, status: str):
        # Update task status in the queue
        pass

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue, status_tracker: 'StatusTracker', retry_mechanism: 'RetryMechanism'):
        super().__init__()
        self.task_queue = task_queue
        self.status_tracker = status_tracker
        self.retry_mechanism = retry_mechanism

    def consume_task(self):
        task = self.task_queue.get_task()
        if task:
            self.execute_task(task)

    def execute_task(self, task: Task):
        try:
            task.callback()
            self.status_tracker.update_task_status(task.id, "completed")
        except Exception as e:
            self.status_tracker.update_task_status(task.id, "failed")
            self.retry_mechanism.retry_task(task, self.task_queue)

    def run(self):
        while True:
            self.consume_task()

class StatusTracker:
    def __init__(self):
        self.status = {}
        self.lock = threading.Lock()

    def get_task_status(self, task_id: int):
        with self.lock:
            return self.status.get(task_id, "pending")

    def update_task_status(self, task_id: int, status: str):
        with self.lock:
            self.status[task_id] = status

class RetryMechanism:
    def __init__(self, max_retries: int):
        self.max_retries = max_retries

    def retry_task(self, task: Task, task_queue: TaskQueue):
        if task.max_retries > 0:
            task.max_retries -= 1
            task_queue.add_task(task)

def main():
    task_queue = TaskQueue()
    status_tracker = StatusTracker()
    retry_mechanism = RetryMechanism(3)

    # Add tasks to the task queue
    task_queue.add_task(Task(1, 1, lambda: print("Task 1 executed")))
    task_queue.add_task(Task(2, 2, lambda: print("Task 2 executed")))

    # Create and start worker threads
    worker_threads = []
    for _ in range(5):
        worker_thread = WorkerThread(task_queue, status_tracker, retry_mechanism)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Wait for tasks to complete
    while True:
        if not task_queue.queue:
            break

    # Stop worker threads
    for worker_thread in worker_threads:
        worker_thread.join()

if __name__ == "__main__":
    main()

**Architecture Document: In-Memory Task Queue**

**Overview**
----------

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status and providing retry mechanisms. The system will ensure a graceful shutdown when required.

**Components**
------------

1. **Task**: Represents a unit of work with a priority and a callback function.
2. **Task Queue**: Manages tasks in memory, providing methods for adding, consuming, and tracking tasks.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of tasks (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable maximum number of attempts.

**Module Decomposition**
----------------------

The system will be decomposed into the following modules:

1. `task.py`: Defines the `Task` class.
2. `task_queue.py`: Implements the `TaskQueue` class using a priority queue.
3. `worker.py`: Defines the `WorkerThread` class with improved error handling and logging.
4. `status_tracker.py`: Implements the `TaskStatusTracker` class with task history and statistics.
5. `retry_mechanism.py`: Defines the `RetryMechanism` class with configurable retry policies and exponential backoff.
6. `main.py`: Demonstrates the usage of the task queue system.

**Interfaces**
-------------

1. `TaskQueue`:
	* `add_task(task: Task)`: Adds a task to the queue.
	* `get_task()`: Retrieves a task from the queue.
	* `update_task_status(task_id: int, status: str)`: Updates the status of a task.
2. `WorkerThread`:
	* `consume_task()`: Consumes a task from the task queue.
	* `execute_task(task: Task)`: Executes a task.
3. `TaskStatusTracker`:
	* `get_task_status(task_id: int)`: Retrieves the status of a task.
	* `update_task_status(task_id: int, status: str)`: Updates the status of a task.
4. `RetryMechanism`:
	* `retry_task(task: Task)`: Retries a task if it has failed.

**Data Flow**
------------

1. A task is added to the task queue using the `add_task` method.
2. A worker thread consumes a task from the task queue using the `get_task` method.
3. The worker thread executes the task using the `execute_task` method.
4. The task status is updated using the `update_task_status` method.
5. If a task fails, the retry mechanism is triggered using the `retry_task` method.
