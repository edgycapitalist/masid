## Introduction

This document outlines the design for an in-memory task queue in Python. The task queue will support adding tasks with priorities, allow worker threads to consume tasks, provide task status tracking, support task retry with configurable max retries, and include graceful shutdown.

## System Components

The task queue system will consist of the following components:

1. **Task**: Represents a single task to be executed. Each task will have a unique ID, priority, and a callable function to be executed.
2. **Task Queue**: The main component responsible for storing and managing tasks. It will provide methods for adding tasks, consuming tasks, and tracking task status.
3. **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
4. **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).

## System Design

The task queue system will be designed as follows:

1. **Task Queue**: The task queue will be implemented as a priority queue using the `heapq` module in Python. This will ensure that tasks with higher priorities are executed first.
2. **Task Status Tracker**: The task status tracker will be implemented as a dictionary where each task ID is mapped to its current status.
3. **Worker Thread**: Worker threads will be implemented using the `threading` module in Python. Each worker thread will continuously consume tasks from the task queue and execute them.
4. **Task Retry**: Task retry will be implemented using a configurable max retries parameter. If a task fails, it will be retried up to the max retries limit.

## System Flow

The system flow will be as follows:

1. **Add Task**: A task is added to the task queue with its priority and callable function.
2. **Consume Task**: A worker thread consumes a task from the task queue and executes it.
3. **Track Task Status**: The task status tracker updates the status of the task to "running".
4. **Task Completion**: If the task completes successfully, the task status tracker updates the status to "completed".
5. **Task Failure**: If the task fails, the task status tracker updates the status to "failed" and the task is retried up to the max retries limit.
6. **Graceful Shutdown**: When the system is shut down, all worker threads are stopped and any remaining tasks in the task queue are discarded.

import heapq
import threading
import time
import logging

class Task:
    def __init__(self, task_id, priority, func, max_retries=3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = "pending"

    def execute(self):
        try:
            self.func()
            self.status = "completed"
        except Exception as e:
            self.status = "failed"
            self.retries += 1
            if self.retries < self.max_retries:
                raise

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.status_tracker = {}
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))
            self.status_tracker[task.task_id] = task

    def consume_task(self):
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

    def track_status(self, task_id, status):
        with self.lock:
            self.status_tracker[task_id].status = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue

    def run(self):
        while True:
            task = self.task_queue.consume_task()
            if task is None:
                break
            self.task_queue.track_status(task.task_id, "running")
            try:
                task.execute()
            except Exception as e:
                logging.error(f"Task {task.task_id} failed: {e}")
                if task.retries < task.max_retries:
                    self.task_queue.add_task(task)
            finally:
                self.task_queue.track_status(task.task_id, task.status)

def main():
    task_queue = TaskQueue()
    worker_threads = []

    # Add tasks to the task queue
    task1 = Task(1, 1, lambda: print("Task 1 executed"))
    task2 = Task(2, 2, lambda: print("Task 2 executed"))
    task_queue.add_task(task1)
    task_queue.add_task(task2)

    # Start worker threads
    for _ in range(2):
        worker_thread = WorkerThread(task_queue)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Wait for worker threads to finish
    for worker_thread in worker_threads:
        worker_thread.join()

if __name__ == "__main__":
    main()
