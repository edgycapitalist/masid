### Overview

The in-memory task queue is designed to manage tasks with priorities, allow worker threads to consume tasks, track task status, and support task retry with configurable max retries. The system will also include a mechanism for graceful shutdown.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
2. **Task Queue**: A data structure that stores tasks in a priority order.
3. **Worker Thread**: A thread that consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: A component that tracks the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: A component that handles task retry with configurable max retries.

### System Design

#### Task Queue

* The task queue will be implemented using a priority queue data structure (e.g., `heapq` in Python).
* Tasks will be added to the queue with their priority and a unique ID.

#### Worker Thread

* Worker threads will be implemented using Python's `threading` module.
* Each worker thread will consume tasks from the task queue, execute them, and update the task status.

#### Task Status Tracker

* The task status tracker will be implemented using a dictionary that maps task IDs to their current status.
* The status will be updated by the worker threads as they execute tasks.

#### Retry Mechanism

* The retry mechanism will be implemented using a separate thread that monitors the task status tracker.
* If a task fails, the retry mechanism will check if the max retries have been reached. If not, it will add the task back to the task queue with a delay.

#### Shutdown Mechanism

* The shutdown mechanism will be implemented using a separate thread that monitors a shutdown flag.
* When the shutdown flag is set, the system will stop adding new tasks to the queue and wait for all running tasks to complete.

        while True:

            if task:

        """
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        """
        Adds a task to the queue with its priority and a unique ID.
        
        Args:
        task (Task): Task to be added to the queue.
        """
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

        """
        Retrieves a task from the queue.
        
        Returns:
        Task: Retrieved task.
        """
        with self.lock:
            if self.queue:
                return heapq.heappop(self.queue)[1]
            return None

        Returns:
        Task: Next task in the queue.
        """
        with self.lock:
            if self.queue:
                return self.queue[0][1]
            return None

        with self.lock:
