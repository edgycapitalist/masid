**Architecture Document: In-Memory Task Queue**
=============================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system provides task retry with configurable max retries and supports graceful shutdown.

### Components

1. **Task**: Represents a unit of work with a unique ID, priority, and retry count.
2. **Task Queue**: A data structure that stores tasks in memory, sorted by priority.
3. **Worker Thread**: Consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable max retry count.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task: Task)`: Adds a task to the queue.
	* `get_task()`: Retrieves the next task from the queue.
2. **Worker Thread Interface**:
	* `start()`: Starts the worker thread.
	* `stop()`: Stops the worker thread.
3. **Task Status Tracker Interface**:
	* `update_status(task_id: int, status: str)`: Updates the status of a task.

### Data Flow

1. **Task Submission**:
	* A client submits a task to the task queue using the `add_task` method.
	* The task is added to the queue and sorted by priority.
2. **Task Consumption**:
	* A worker thread retrieves the next task from the queue using the `get_task` method.
	* The worker thread executes the task and updates its status using the `update_status` method.
3. **Task Retry**:
	* If a task fails, the retry mechanism is triggered.
	* The task is retried up to the max retry count, and its status is updated accordingly.

### Technology Choices

* **Python**: The programming language used for implementation.
* **Threading**: The library used for creating and managing worker threads.
* **Queue**: The data structure used for storing tasks in memory.

```python
import threading
import queue
import time

class Task:
    def __init__(self, id, priority, func, *args, **kwargs):
        self.id = id
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.retry_count = 0
        self.max_retries = 3

    def execute(self):
        try:
            self.func(*self.args, **self.kwargs)
            return True
        except Exception as e:
            print(f"Task {self.id} failed: {e}")
            return False

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))

    def get_task(self):
        if self.queue.empty():
            return None
        return self.queue.get()[1]

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(1)
                continue
            self.task_status_tracker.update_status(task.id, "running")
            if task.execute():
                self.task_status_tracker.update_status(task.id, "completed")
            else:
                if task.retry_count < task.max_retries:
                    task.retry_count += 1
                    self.task_queue.add_task(task)
                    self.task_status_tracker.update_status(task.id, "pending")
                else:
                    self.task_status_tracker.update_status(task.id, "failed")

class TaskStatusTracker:
    def __init__(self):
        self.status = {}

    def update_status(self, task_id, status):
        self.status[task_id] = status

def example_task():
    print("Example task executed")

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()
    worker_thread = WorkerThread(task_queue, task_status_tracker)
    worker_thread.start()

    task = Task(1, 1, example_task)
    task_queue.add_task(task)

    time.sleep(5)
    worker_thread.running = False
    worker_thread.join()

if __name__ == "__main__":
    main()

2. **Task Queue**: A data structure that stores tasks in memory, sorted by priority.
3. **Worker Thread**: Consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable max retry count.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task: Task)`: Adds a task to the queue.
	* `get_task()`: Retrieves the next task from the queue.
2. **Worker Thread Interface**:
	* `start()`: Starts the worker thread.
	* `stop()`: Stops the worker thread.
3. **Task Status Tracker Interface**:
	* `update_status(task_id: int, status: str)`: Updates the status of a task.

### Data Flow

1. **Task Submission**:
	* A client submits a task to the task queue using the `add_task` method.
	* The task is added to the queue and sorted by priority.
2. **Task Consumption**:
	* A worker thread retrieves the next task from the queue using the `get_task` method.
	* The worker thread executes the task and updates its status using the `update_status` method.
3. **Task Retry**:
	* If a task fails, the retry mechanism is triggered.
	* The task is retried up to the max retry count, and its status is updated accordingly.

import threading
import queue
import time
import unittest

class Task:
    def __init__(self, id, priority, func, *args, **kwargs):
        self.id = id
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.retry_count = 0
        self.max_retries = 3
        self.timeout = 10  # add a timeout attribute

    def execute(self):
        try:
            self.func(*self.args, **self.kwargs)
            return True
        except Exception as e:
            print(f"Task {self.id} failed: {e}")
            return False

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.lock = threading.Lock()
        self.max_size = 100  # add a max size attribute

    def add_task(self, task):
        with self.lock:
            if self.queue.qsize() < self.max_size:
                self.queue.put((task.priority, task))
            else:
                print("Task queue is full")

    def get_task(self):
        with self.lock:
            if self.queue.empty():
                return None
            return self.queue.get()[1]

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(1)
                continue
            self.task_status_tracker.update_status(task.id, "running")
            try:
                if task.execute():
                    self.task_status_tracker.update_status(task.id, "completed")
                else:
                    if task.retry_count < task.max_retries:
                        task.retry_count += 1
                        self.task_queue.add_task(task)
                        self.task_status_tracker.update_status(task.id, "pending")
                    else:
                        self.task_status_tracker.update_status(task.id, "failed")
            except Exception as e:
                print(f"Error executing task {task.id}: {e}")

class TaskStatusTracker:
    def __init__(self):
        self.status = {}
        self.lock = threading.Lock()

    def update_status(self, task_id, status):
        with self.lock:
            self.status[task_id] = status

def example_task():
    print("Example task executed")

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()
    worker_thread = WorkerThread(task_queue, task_status_tracker)
    worker_thread.start()

    task = Task(1, 1, example_task)
    task_queue.add_task(task)

    time.sleep(5)
    worker_thread.running = False
    worker_thread.join()

if __name__ == "__main__":
    main()

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system provides task retry with configurable max retries and supports graceful shutdown.

### Components

1. **Task**: Represents a unit of work with a unique ID, priority, and retry count.
2. **Task Queue**: A data structure that stores tasks in memory, sorted by priority.
3. **Worker Thread**: Consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable max retry count.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task: Task)`: Adds a task to the queue.
	* `get_task()`: Retrieves the next task from the queue.
2. **Worker Thread Interface**:
	* `start()`: Starts the worker thread.
	* `stop()`: Stops the worker thread.
3. **Task Status Tracker Interface**:
	* `update_status(task_id: int, status: str)`: Updates the status of a task.

### Data Flow

1. **Task Submission**:
	* A client submits a task to the task queue using the `add_task` method.
	* The task is added to the queue and sorted by priority.
2. **Task Consumption**:
	* A worker thread retrieves the next task from the queue using the `get_task` method.
	* The worker thread executes the task and updates its status using the `update_status` method.
3. **Task Retry**:
	* If a task fails, the retry mechanism is triggered.
	* The task is retried up to the max retry count, and its status is updated accordingly.
