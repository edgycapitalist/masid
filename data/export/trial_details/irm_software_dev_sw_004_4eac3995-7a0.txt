## Overview

The in-memory task queue system is designed to manage tasks with varying priorities, track their status, and provide a mechanism for worker threads to consume tasks. The system will also support task retry with a configurable maximum number of retries and include a graceful shutdown mechanism.

## Components

1. **Task**: Represents a unit of work with a unique identifier, priority, and a callable function to execute.
2. **Task Queue**: An in-memory data structure that stores tasks and provides methods for adding, retrieving, and updating tasks.
3. **Worker Thread**: A thread that consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: A component that tracks the status of tasks (pending, running, completed, failed).

## System Flow

1. **Task Submission**:
	* A task is added to the task queue with a specified priority.
	* The task is assigned a unique identifier and its status is set to "pending".
2. **Task Consumption**:
	* A worker thread retrieves a task from the task queue based on its priority.
	* The task's status is updated to "running".
	* The worker thread executes the task.
	* If the task completes successfully, its status is updated to "completed".
	* If the task fails, its status is updated to "failed" and it is retried if the maximum number of retries has not been exceeded.
3. **Task Retry**:
	* If a task fails, it is retried up to a configurable maximum number of times.
	* After each retry, the task's status is updated to "failed" and the retry count is incremented.
4. **Gracious Shutdown**:
	* When the system is shut down, all worker threads are stopped and the task queue is cleared.

## Data Structures

1. **Task Queue**: A priority queue implemented using a heap data structure.
2. **Task Status Tracker**: A dictionary that maps task identifiers to their corresponding status.

## Interfaces

1. **Task Queue Interface**:
	* `add_task(task)`: Adds a task to the task queue.
	* `get_task()`: Retrieves a task from the task queue based on its priority.
	* `update_task_status(task_id, status)`: Updates the status of a task.
2. **Worker Thread Interface**:
	* `consume_task()`: Retrieves a task from the task queue and executes it.
	* `update_task_status(task_id, status)`: Updates the status of a task.

import threading
import heapq
from dataclasses import dataclass
from typing import Callable, Dict

@dataclass
class Task:
    """Represents a task with a unique identifier, priority, and a callable function to execute."""
    id: int
    priority: int
    func: Callable
    max_retries: int = 3

class TaskQueue:
    """In-memory task queue that supports adding tasks with priorities and retrieving tasks based on their priority."""
    def __init__(self):
        self.tasks = []
        self.task_status: Dict[int, str] = {}
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        """Adds a task to the task queue."""
        with self.lock:
            heapq.heappush(self.tasks, (task.priority, task))
            self.task_status[task.id] = "pending"

    def get_task(self) -> Task:
        """Retrieves a task from the task queue based on its priority."""
        with self.lock:
            if not self.tasks:
                return None
            _, task = heapq.heappop(self.tasks)
            return task

    def update_task_status(self, task_id: int, status: str):
        """Updates the status of a task."""
        with self.lock:
            self.task_status[task_id] = status

class WorkerThread(threading.Thread):
    """Worker thread that consumes tasks from the task queue and executes them."""
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            self.task_queue.update_task_status(task.id, "running")
            try:
                task.func()
                self.task_queue.update_task_status(task.id, "completed")
            except Exception as e:
                self.task_queue.update_task_status(task.id, "failed")
                if task.max_retries > 0:
                    self.task_queue.add_task(Task(task.id, task.priority, task.func, task.max_retries - 1))
                else:
                    print(f"Task {task.id} failed after {task.max_retries} retries")

def main():
    task_queue = TaskQueue()

    # Add tasks to the task queue
    task_queue.add_task(Task(1, 1, lambda: print("Task 1 executed")))
    task_queue.add_task(Task(2, 2, lambda: print("Task 2 executed")))

    # Create and start worker threads
    worker_threads = [WorkerThread(task_queue) for _ in range(2)]
    for thread in worker_threads:
        thread.start()

    # Wait for worker threads to finish
    for thread in worker_threads:
        thread.join()

if __name__ == "__main__":
    main()

## Overview

The in-memory task queue system is designed to manage tasks with varying priorities, track their status, and provide a mechanism for worker threads to consume tasks. The system will also support task retry with a configurable maximum number of retries and include a graceful shutdown mechanism.

## Components

1. **Task**: Represents a unit of work with a unique identifier, priority, and a callable function to execute.
2. **Task Queue**: An in-memory data structure that stores tasks and provides methods for adding, retrieving, and updating tasks.
3. **Worker Thread**: A thread that consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: A component that tracks the status of tasks (pending, running, completed, failed).

## System Flow

1. **Task Submission**:
	* A task is added to the task queue with a specified priority.
	* The task is assigned a unique identifier and its status is set to "pending".
2. **Task Consumption**:
	* A worker thread retrieves a task from the task queue based on its priority.
	* The task's status is updated to "running".
	* The worker thread executes the task.
	* If the task completes successfully, its status is updated to "completed".
	* If the task fails, its status is updated to "failed" and it is retried if the maximum number of retries has not been exceeded.
3. **Task Retry**:
	* If a task fails, it is retried up to a configurable maximum number of times.
	* After each retry, the task's status is updated to "failed" and the retry count is incremented.
4. **Gracious Shutdown**:
	* When the system is shut down, all worker threads are stopped and the task queue is cleared.

## Data Structures

1. **Task Queue**: A priority queue implemented using a heap data structure.
2. **Task Status Tracker**: A dictionary that maps task identifiers to their corresponding status.

## Interfaces

1. **Task Queue Interface**:
	* `add_task(task)`: Adds a task to the task queue.
	* `get_task()`: Retrieves a task from the task queue based on its priority.
	* `update_task_status(task_id, status)`: Updates the status of a task.
2. **Worker Thread Interface**:
	* `consume_task()`: Retrieves a task from the task queue and executes it.

import threading
import heapq
from dataclasses import dataclass
from typing import Callable, Dict
import logging
import time

# Initialize logger
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

@dataclass
class Task:
    """Represents a task with a unique identifier, priority, and a callable function to execute."""
    id: int
    priority: int
    func: Callable
    max_retries: int = 3
    retry_count: int = 0

class TaskQueue:
    """In-memory task queue that supports adding tasks with priorities and retrieving tasks based on their priority."""
    def __init__(self):
        self.tasks = []
        self.task_status: Dict[int, str] = {}
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        """Adds a task to the task queue."""
        with self.lock:
            heapq.heappush(self.tasks, (task.priority, task))
            self.task_status[task.id] = "pending"
            logger.info(f"Task {task.id} added to the task queue")

    def get_task(self) -> Task:
        """Retrieves a task from the task queue based on its priority."""
        with self.lock:
            if not self.tasks:
                return None
            _, task = heapq.heappop(self.tasks)
            return task

    def update_task_status(self, task_id: int, status: str):
        """Updates the status of a task."""
        with self.lock:
            self.task_status[task_id] = status
            logger.info(f"Task {task_id} status updated to {status}")

    def remove_task(self, task_id: int):
        """Removes a task from the task queue."""
        with self.lock:
            self.tasks = [(priority, task) for priority, task in self.tasks if task.id != task_id]
            if task_id in self.task_status:
                del self.task_status[task_id]

    def clear_tasks(self):
        """Clears all tasks from the task queue."""
        with self.lock:
            self.tasks = []
            self.task_status = {}

class WorkerThread(threading.Thread):
    """Worker thread that consumes tasks from the task queue and executes them."""
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.stop_event = threading.Event()

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(0.1)  # Wait for 0.1 seconds before checking again
                continue
            self.task_queue.update_task_status(task.id, "running")
            try:
                task.func()
                self.task_queue.update_task_status(task.id, "completed")
            except Exception as e:
                self.task_queue.update_task_status(task.id, "failed")
                logger.error(f"Task {task.id} failed with error: {str(e)}")
                if task.retry_count < task.max_retries:
                    task.retry_count += 1
                    logger.info(f"Task {task.id} failed, retrying...")
                    self.task_queue.add_task(task)
                else:
                    logger.info(f"Task {task.id} failed after {task.max_retries} retries")

    def stop(self):
        self.stop_event.set()

def main():
    task_queue = TaskQueue()

    # Add tasks to the task queue
    task_queue.add_task(Task(1, 1, lambda: print("Task 1 executed")))
    task_queue.add_task(Task(2, 2, lambda: print("Task 2 executed")))

    # Create and start worker threads
    worker_threads = [WorkerThread(task_queue) for _ in range(2)]
    for thread in worker_threads:
        thread.start()

    # Wait for 5 seconds
    time.sleep(5)

    # Stop worker threads
    for thread in worker_threads:
        thread.stop()
        thread.join()
