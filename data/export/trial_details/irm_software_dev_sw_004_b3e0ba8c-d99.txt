**Architecture Document: In-Memory Task Queue**
=====================================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks and track their status. The system provides task retry with configurable max retries and supports graceful shutdown.

### Components

1. **Task**: Represents a unit of work with a unique ID, priority, and a callable function.
2. **Task Queue**: A data structure that stores tasks in memory, allowing for efficient addition and retrieval of tasks based on priority.
3. **Worker Thread**: A thread that consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: A component that tracks the status of tasks, including pending, running, completed, and failed.
5. **Retry Mechanism**: A component that handles task retry with configurable max retries.

### System Flow

1. **Task Addition**:
	* A task is added to the task queue with a specified priority.
	* The task is stored in the task queue data structure.
2. **Task Consumption**:
	* A worker thread retrieves the highest-priority task from the task queue.
	* The worker thread executes the task and updates its status.
3. **Task Status Tracking**:
	* The task status tracker updates the task status based on the worker thread's execution result.
	* The task status is stored in the task status tracker.
4. **Task Retry**:
	* If a task fails, the retry mechanism checks the max retries configuration.
	* If the max retries limit is not exceeded, the task is retried.

### Implementation Details

* The task queue data structure will be implemented using a priority queue (e.g., `heapq` in Python).
* The task status tracker will use a dictionary to store task IDs and their corresponding statuses.
* The retry mechanism will use a configurable max retries setting to determine when to retry a task.

### Implementation Code
```python
import threading
import heapq
from enum import Enum
from typing import Callable, Dict

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.status = TaskStatus.PENDING
        self.retries = 0

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self) -> Task:
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue, max_retries: int):
        super().__init__()
        self.task_queue = task_queue
        self.max_retries = max_retries

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            try:
                task.status = TaskStatus.RUNNING
                task.func()
                task.status = TaskStatus.COMPLETED
            except Exception as e:
                task.status = TaskStatus.FAILED
                task.retries += 1
                if task.retries < self.max_retries:
                    self.task_queue.add_task(task)

class TaskStatusTracker:
    def __init__(self):
        self.statuses: Dict[str, TaskStatus] = {}

    def update_status(self, task: Task):
        self.statuses[task.task_id] = task.status

def main():
    task_queue = TaskQueue()
    worker_thread = WorkerThread(task_queue, max_retries=3)
    worker_thread.start()

    def example_task():
        print("Example task executed")

    task = Task("example_task", 1, example_task)
    task_queue.add_task(task)

    worker_thread.join()

if __name__ == "__main__":
```
