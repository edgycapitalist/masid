**Architecture Document: In-Memory Task Queue**
=====================================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, track task status, and support task retry with configurable max retries. The system will utilize a thread-safe queue to handle task addition and consumption, ensuring efficient and reliable task processing.

### Components

1. **Task**: Represents a single task to be executed, containing:
	* `id`: Unique task identifier
	* `priority`: Task priority (integer value, higher is more important)
	* `max_retries`: Maximum number of retries allowed
	* `status`: Current task status (pending, running, completed, failed)
2. **Task Queue**: A thread-safe queue that stores tasks to be executed, ordered by priority.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates task status.
4. **Task Executor**: Responsible for executing tasks and handling retries.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task)`: Adds a task to the queue
	* `get_task()`: Retrieves the next task from the queue
2. **Task Executor Interface**:
	* `execute_task(task)`: Executes a task and updates its status
	* `retry_task(task)`: Retries a failed task
3. **Worker Thread Interface**:
	* `start()`: Starts the worker thread
	* `stop()`: Stops the worker thread

### Data Flow

1. **Task Addition**:
	* Client adds a task to the task queue using the `add_task` method.
	* Task is stored in the queue, ordered by priority.
2. **Task Consumption**:
	* Worker thread retrieves the next task from the queue using `get_task`.
	* Task is executed by the task executor.
	* Task status is updated based on the execution result.
3. **Task Retry**:
	* If a task fails, the task executor retries the task up to the maximum allowed retries.
	* If all retries fail, the task is marked as failed.

### Technology Choices

* Python 3.x as the programming language
* `threading` module for thread-safe queue implementation
* `queue` module for thread-safe queue operations

```python
import threading
import queue
import time

class Task:
    def __init__(self, id, priority, max_retries, func):
        self.id = id
        self.priority = priority
        self.max_retries = max_retries
        self.status = "pending"
        self.func = func
        self.retries = 0

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))

    def get_task(self):
        if self.queue.empty():
            return None
        return self.queue.get()[1]

class TaskExecutor:
    def __init__(self):
        pass

    def execute_task(self, task):
        try:
            task.func()
            task.status = "completed"
        except Exception as e:
            task.status = "failed"
            self.retry_task(task, e)

    def retry_task(self, task, exception):
        if task.retries < task.max_retries:
            task.retries += 1
            task.status = "pending"
            # Simulate a delay before retrying
            time.sleep(1)
            self.execute_task(task)
        else:
            task.status = "failed"

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_executor):
        super().__init__()
        self.task_queue = task_queue
        self.task_executor = task_executor
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                self.task_executor.execute_task(task)
            else:
                # Sleep for a short period to avoid busy-waiting
                time.sleep(0.1)

    def stop(self):
        self.running = False

# Example usage
def example_task():
    print("Task executed")

task_queue = TaskQueue()
task_executor = TaskExecutor()
worker_thread = WorkerThread(task_queue, task_executor)

task = Task(1, 1, 3, example_task)
task_queue.add_task(task)

worker_thread.start()
time.sleep(2)
worker_thread.stop()
```

### Overview

The in-memory task queue is designed to manage tasks with priorities, track task status, and support task retry with configurable max retries. The system will utilize a thread-safe queue to handle task addition and consumption, ensuring efficient and reliable task processing.

### Components

1. **Task**: Represents a single task to be executed, containing:
	* `id`: Unique task identifier
	* `priority`: Task priority (integer value, higher is more important)
	* `max_retries`: Maximum number of retries allowed
	* `status`: Current task status (pending, running, completed, failed)
	* `description`: Task description
	* `created_at`: Task creation timestamp
2. **Task Queue**: A thread-safe queue that stores tasks to be executed, ordered by priority.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates task status.
4. **Task Executor**: Responsible for executing tasks and handling retries.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task)`: Adds a task to the queue
	* `get_task()`: Retrieves the next task from the queue
2. **Task Executor Interface**:
	* `execute_task(task)`: Executes a task and updates its status
	* `retry_task(task)`: Retries a failed task
3. **Worker Thread Interface**:
	* `start()`: Starts the worker thread
	* `stop()`: Stops the worker thread

### Data Flow

1. **Task Addition**:
	* Client adds a task to the task queue using the `add_task` method.
	* Task is stored in the queue, ordered by priority.
2. **Task Consumption**:
	* Worker thread retrieves the next task from the queue using `get_task`.
	* Task is executed by the task executor.
	* Task status is updated based on the execution result.
3. **Task Retry**:
	* If a task fails, the task executor retries the task up to the maximum allowed retries.
	* If all retries fail, the task is marked as failed.

```python
import threading
import queue
import time
import logging
from loguru import logger

class Task:
    def __init__(self, id, priority, max_retries, func, description=None):
        self.id = id
        self.priority = priority
        self.max_retries = max_retries
        self.status = "pending"
        self.func = func
        self.retries = 0
        self.description = description
        self.created_at = time.time()

    def __str__(self):
        return f"Task {self.id}: {self.description}, Priority {self.priority}, Status {self.status}"

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))
        logger.info(f"Task added: {task}")

    def get_task(self):
        if self.queue.empty():
            return None
        return self.queue.get()[1]

class TaskExecutor:
    def __init__(self):
        pass

    def execute_task(self, task):
        try:
            task.func()
            task.status = "completed"
            logger.info(f"Task completed: {task}")
        except Exception as e:
            task.status = "failed"
            logger.error(f"Task failed: {task}, Error: {str(e)}")
            self.retry_task(task, e)

    def retry_task(self, task, exception):
        if task.retries < task.max_retries:
            task.retries += 1
            task.status = "pending"
            logger.info(f"Task {task.id} retrying ({task.retries}/{task.max_retries})")
            # Simulate a delay before retrying
            time.sleep(1)
            self.execute_task(task)
        else:
            task.status = "failed"
            logger.error(f"Task {task.id} failed after {task.max_retries} retries")

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_executor):
        super().__init__()
        self.task_queue = task_queue
        self.task_executor = task_executor
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                self.task_executor.execute_task(task)
            else:
                # Sleep for a short period to avoid busy-waiting
                time.sleep(0.1)

    def stop(self):
        self.running = False
```

**Architecture Document: In-Memory Task Queue**
=====================================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, track task status, and support task retry with configurable max retries. The system will utilize a thread-safe queue to handle task addition and consumption, ensuring efficient and reliable task processing.

### Components

1. **Task**: Represents a single task to be executed, containing:
	* `id`: Unique task identifier
	* `priority`: Task priority (integer value, higher is more important)
	* `max_retries`: Maximum number of retries allowed
	* `status`: Current task status (pending, running, completed, failed)
	* `description`: Task description
	* `created_at`: Task creation timestamp
2. **Task Queue**: A thread-safe queue that stores tasks to be executed, ordered by priority.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates task status.
4. **Task Executor**: Responsible for executing tasks and handling retries.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task)`: Adds a task to the queue
	* `get_task()`: Retrieves the next task from the queue
2. **Task Executor Interface**:
	* `execute_task(task)`: Executes a task and updates its status
	* `retry_task(task)`: Retries a failed task
3. **Worker Thread Interface**:

```python
        logger.info(f"Task added: {task}")

        if self.queue.empty():

        return self.queue.get()[1]

    def __init__(self):
```
