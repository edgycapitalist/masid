### Overview

The in-memory task queue is designed to manage tasks with priorities, support worker threads, track task status, and provide task retry with configurable max retries. The system will ensure graceful shutdown to prevent data loss.

### Module Decomposition

The task queue system will consist of the following modules:

1. **Task**: Represents a single task with attributes such as `id`, `priority`, `status`, and `max_retries`.
2. **TaskQueue**: Manages the in-memory task queue, providing methods for adding tasks, consuming tasks, and tracking task status.
3. **WorkerThread**: Responsible for consuming tasks from the task queue and executing them.
4. **TaskExecutor**: Executes tasks and handles task retries.
5. **ShutdownManager**: Manages the shutdown process, ensuring that all tasks are completed or terminated before shutting down.

### Interfaces

The following interfaces will be used to interact between modules:

1. **TaskQueueInterface**: Defines methods for adding tasks, consuming tasks, and tracking task status.
2. **WorkerThreadInterface**: Defines methods for starting and stopping worker threads.
3. **TaskExecutorInterface**: Defines methods for executing tasks and handling task retries.
4. **ShutdownManagerInterface**: Defines methods for initiating and managing shutdown.

### Data Flow

The data flow between modules will be as follows:

1. **Task** -> **TaskQueue**: Tasks are added to the task queue through the `TaskQueueInterface`.
2. **TaskQueue** -> **WorkerThread**: Worker threads consume tasks from the task queue through the `TaskQueueInterface`.
3. **WorkerThread** -> **TaskExecutor**: Worker threads execute tasks through the `TaskExecutorInterface`.
4. **TaskExecutor** -> **TaskQueue**: Task execution results are reported back to the task queue through the `TaskQueueInterface`.
5. **ShutdownManager** -> **TaskQueue**: Shutdown manager initiates shutdown, and the task queue ensures that all tasks are completed or terminated.

### Technology Choices

The following technologies will be used:

1. **Python**: The primary programming language for the task queue system.
2. **Threading**: For implementing worker threads.
3. **Queue**: For implementing the in-memory task queue.
4. **Enum**: For representing task status (pending, running, completed, failed).

import threading
import queue
from enum import Enum

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, max_retries):
        self.id = id
        self.priority = priority
        self.status = TaskStatus.PENDING
        self.max_retries = max_retries
        self.retries = 0

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))

    def consume_task(self):
        if not self.queue.empty():
            return self.queue.get()
        return None

    def update_task_status(self, task, status):
        task.status = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_executor):
        super().__init__()
        self.task_queue = task_queue
        self.task_executor = task_executor

    def run(self):
        while True:
            task = self.task_queue.consume_task()
            if task is None:
                break
            self.task_executor.execute_task(task)

class TaskExecutor:
    def __init__(self, task_queue):
        self.task_queue = task_queue

    def execute_task(self, task):
        try:
            # Execute task logic here
            print(f"Executing task {task.id}")
            self.task_queue.update_task_status(task, TaskStatus.COMPLETED)
        except Exception as e:
            if task.retries < task.max_retries:
                task.retries += 1
                self.task_queue.add_task(task)
            else:
                self.task_queue.update_task_status(task, TaskStatus.FAILED)

class ShutdownManager:

**Improved Architecture Document: In-Memory Task Queue**
=====================================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, support worker threads, track task status, and provide task retry with configurable max retries. The system will ensure graceful shutdown to prevent data loss.

### Module Decomposition

The task queue system will consist of the following modules:

1. **Task**: Represents a single task with attributes such as `id`, `priority`, `status`, and `max_retries`.
2. **TaskQueue**: Manages the in-memory task queue, providing methods for adding tasks, consuming tasks, and tracking task status.
3. **WorkerThread**: Responsible for consuming tasks from the task queue and executing them.
4. **TaskExecutor**: Executes tasks and handles task retries.
5. **ShutdownManager**: Manages the shutdown process, ensuring that all tasks are completed or terminated before shutting down.

### Interfaces

The following interfaces will be used to interact between modules:

1. **TaskQueueInterface**: Defines methods for adding tasks, consuming tasks, and tracking task status.
2. **WorkerThreadInterface**: Defines methods for starting and stopping worker threads.
3. **TaskExecutorInterface**: Defines methods for executing tasks and handling task retries.
4. **ShutdownManagerInterface**: Defines methods for initiating and managing shutdown.

### Data Flow

The data flow between modules will be as follows:

1. **Task** -> **TaskQueue**: Tasks are added to the task queue through the `TaskQueueInterface`.
2. **TaskQueue** -> **WorkerThread**: Worker threads consume tasks from the task queue through the `TaskQueueInterface`.
3. **WorkerThread** -> **TaskExecutor**: Worker threads execute tasks through the `TaskExecutorInterface`.
4. **TaskExecutor** -> **TaskQueue**: Task execution results are reported back to the task queue through the `TaskQueueInterface`.
5. **ShutdownManager** -> **TaskQueue**: Shutdown manager initiates shutdown, and the task queue ensures that all tasks are completed or terminated.

### Technology Choices

The following technologies will be used:

1. **Python**: The primary programming language for the task queue system.
2. **Threading**: For implementing worker threads.
3. **Queue**: For implementing the in-memory task queue.
4. **Enum**: For representing task status (pending, running, completed, failed).

import threading
import queue
from enum import Enum
import logging
import time

# Set up logging
logging.basicConfig(level=logging.INFO)

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, max_retries, func, *args, **kwargs):
        self.id = id
        self.priority = priority
        self.status = TaskStatus.PENDING
        self.max_retries = max_retries
        self.retries = 0
        self.func = func
        self.args = args
        self.kwargs = kwargs

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.queue.put((task.priority, task))

    def consume_task(self):
        with self.lock:
            if not self.queue.empty():
                return self.queue.get()
            return None

    def update_task_status(self, task, status):
        with self.lock:
            task.status = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_executor):
        super().__init__()
        self.task_queue = task_queue
        self.task_executor = task_executor
        self.stopped = threading.Event()

    def run(self):
        while not self.stopped.is_set():
            task = self.task_queue.consume_task()
            if task is None:
                time.sleep(0.1)  # Sleep for 100ms if queue is empty
                continue
            self.task_executor.execute_task(task)

    def stop(self):
        self.stopped.set()

class TaskExecutor:
    def __init__(self, task_queue):
        self.task_queue = task_queue

    def execute_task(self, task):
        try:
            # Execute task logic here
            logging.info(f"Executing task {task.id}")
            task.func(*task.args, **task.kwargs)
            self.task_queue.update_task_status(task, TaskStatus.COMPLETED)
        except Exception as e:
            logging.error(f"Task {task.id} failed: {e}")
            if task.retries < task.max_retries:
                task.retries += 1
                self.task_queue.add_task(task)
            else:
                self.task_queue.update_task_status(task, TaskStatus.FAILED)

class ShutdownManager:
    def __init__(self, task_queue, worker_threads):
        self.task_queue = task_queue
        self.worker_threads = worker_threads

    def shutdown(self):
        for thread in self.worker_threads:
            thread.stop()
        for thread in self.worker_threads:
            thread.join()
        while not self.task_queue.queue.empty():
            task = self.task_queue.consume_task()
            if task is not None:
                self.task_queue.update_task_status(task, TaskStatus.COMPLETED)

# Example usage
def example_task(name):
    logging.info(f"Hello, {name}!")

task_queue = TaskQueue()
task_executor = TaskExecutor(task_queue)
worker_threads = [WorkerThread(task_queue, task_executor) for _ in range(5)]

task1 = Task(1, 1, 3, example_task, "Alice")
task2 = Task(2, 2, 2, example_task, "Bob")

task_queue.add_task(task1)
task_queue.add_task(task2)

for thread in worker_threads:
    thread.start()

# Simulate task execution

shutdown_manager.shutdown()

### Overview

The in-memory task queue is designed to manage tasks with priorities, support worker threads, track task status, and provide task retry with configurable max retries. The system will ensure graceful shutdown to prevent data loss.

### Module Decomposition

The task queue system will consist of the following modules:

1. **Task**: Represents a single task with attributes such as `id`, `priority`, `status`, and `max_retries`.
2. **TaskQueue**: Manages the in-memory task queue, providing methods for adding tasks, consuming tasks, and tracking task status.
3. **WorkerThread**: Responsible for consuming tasks from the task queue and executing them.
4. **TaskExecutor**: Executes tasks and handles task retries.
5. **ShutdownManager**: Manages the shutdown process, ensuring that all tasks are completed or terminated before shutting down.

### Interfaces

The following interfaces will be used to interact between modules:

1. **TaskQueueInterface**: Defines methods for adding tasks, consuming tasks, and tracking task status.
2. **WorkerThreadInterface**: Defines methods for starting and stopping worker threads.
3. **TaskExecutorInterface**: Defines methods for executing tasks and handling task retries.
4. **ShutdownManagerInterface**: Defines methods for initiating and managing shutdown.

### Data Flow

The data flow between modules will be as follows:

1. **Task** -> **TaskQueue**: Tasks are added to the task queue through the `TaskQueueInterface`.
2. **TaskQueue** -> **WorkerThread**: Worker threads consume tasks from the task queue through the `TaskQueueInterface`.
3. **WorkerThread** -> **TaskExecutor**: Worker threads execute tasks through the `TaskExecutorInterface`.
4. **TaskExecutor** -> **TaskQueue**: Task execution results are reported back to the task queue through the `TaskQueueInterface`.
5. **ShutdownManager** -> **TaskQueue**: Shutdown manager initiates shutdown, and the task queue ensures that all tasks are completed or terminated.

The following technologies will be used:

1. **Python**: The primary programming language for the task queue system.
