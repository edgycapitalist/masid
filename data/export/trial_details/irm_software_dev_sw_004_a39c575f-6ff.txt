**Architecture Document: In-Memory Task Queue**

## Introduction

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The queue supports task retry with configurable max retries and includes a mechanism for graceful shutdown.

## System Components

1. **Task Queue**: The central component responsible for storing and managing tasks.
2. **Task**: Represents a single unit of work with a priority, status, and retry count.
3. **Worker Thread**: Consumes tasks from the queue and executes them.
4. **Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retries with a configurable max retry count.

## System Design

### Task Queue

* The task queue is implemented as a priority queue (heap) to ensure tasks are consumed in order of priority.
* Tasks are stored in a dictionary with their task ID as the key for efficient lookup and status tracking.

### Task

* Each task has a unique task ID, priority, and retry count.
* Tasks are represented as objects with methods for executing the task and updating its status.

### Worker Thread

* Worker threads consume tasks from the queue and execute them.
* Each worker thread runs in a separate thread to ensure concurrent task execution.

### Status Tracker

* The status tracker maintains a dictionary of task IDs to their corresponding status (pending, running, completed, failed).
* The status tracker updates the task status as the task progresses through its lifecycle.

### Retry Mechanism

* The retry mechanism checks if a task has failed and retries it if the max retry count has not been reached.
* The retry mechanism uses an exponential backoff strategy to delay retries.

import threading
import heapq
import time
from enum import Enum

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id, priority, func, max_retries):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retry_count = 0
        self.status = TaskStatus.PENDING

    def execute(self):
        try:
            self.func()
            self.status = TaskStatus.COMPLETED
        except Exception as e:
            self.status = TaskStatus.FAILED
            self.retry_count += 1
            if self.retry_count <= self.max_retries:
                # Retry the task with exponential backoff
                time.sleep(2 ** self.retry_count)
                self.execute()

class TaskQueue:
    def __init__(self):
        self.tasks = {}
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.tasks[task.task_id] = task
            heapq.heappush(self.queue, (task.priority, task.task_id))

    def consume_task(self):
        with self.lock:
            if not self.queue:
                return None
            _, task_id = heapq.heappop(self.queue)
            task = self.tasks[task_id]
            task.status = TaskStatus.RUNNING
            return task

    def update_task_status(self, task_id, status):
        with self.lock:
            task = self.tasks[task_id]
            task.status = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue

    def run(self):
        while True:
            task = self.task_queue.consume_task()
            if task is None:
                break
            task.execute()
            self.task_queue.update_task_status(task.task_id, task.status)

def main():
    task_queue = TaskQueue()

    # Add tasks to the queue
    task1 = Task(1, 1, lambda: print("Task 1 executed"), 3)
    task2 = Task(2, 2, lambda: print("Task 2 executed"), 2)
    task_queue.add_task(task1)
    task_queue.add_task(task2)

    # Start worker threads
    worker1 = WorkerThread(task_queue)
    worker2 = WorkerThread(task_queue)
    worker1.start()
    worker2.start()

    worker1.join()
    worker2.join()
