### Introduction

The in-memory task queue is designed to manage tasks with priorities, track their status, and provide a mechanism for worker threads to consume these tasks. The system will support task retry with a configurable maximum number of retries and include a mechanism for graceful shutdown.

### System Components

2. **Task Queue**: A data structure that stores tasks in a priority order. The task queue is responsible for managing task priorities, status tracking, and task retry.
3. **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
4. **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).

### System Design

#### Worker Thread

The worker thread will be responsible for consuming tasks from the task queue and executing them. The worker thread will use a separate thread to execute tasks to avoid blocking the main thread.

#### Task Status Tracker

The task status tracker will be responsible for tracking the status of each task. The task status tracker will maintain a dictionary to store the status of each task.

#### Task Retry Mechanism

#### Graceful Shutdown

The system will include a mechanism for graceful shutdown. When the system is shut down, all worker threads will be stopped, and any pending tasks will be cancelled.

import heapq
import threading
from enum import Enum

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, callback):
        self.id = id
        self.priority = priority
        self.callback = callback
        self.status = TaskStatus.PENDING
        self.retries = 0

class TaskQueue:
    def __init__(self):
        self.tasks = []
        self.status_tracker = {}

    def add_task(self, task):
        heapq.heappush(self.tasks, (task.priority, task))
        self.status_tracker[task.id] = task.status

    def consume_task(self):
        if not self.tasks:
            return None
        _, task = heapq.heappop(self.tasks)
        task.status = TaskStatus.RUNNING
        self.status_tracker[task.id] = task.status
        return task

    def update_task_status(self, task_id, status):
        if task_id in self.status_tracker:
            self.status_tracker[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue
        self.max_retries = 3

    def run(self):
        while True:
            task = self.task_queue.consume_task()
            if task is None:
                break
            try:
                task.callback()
                task.status = TaskStatus.COMPLETED
                self.task_queue.update_task_status(task.id, task.status)
            except Exception as e:
                task.retries += 1
                if task.retries < self.max_retries:
                    task.status = TaskStatus.PENDING
                    self.task_queue.update_task_status(task.id, task.status)
                    heapq.heappush(self.task_queue.tasks, (task.priority, task))
                else:
                    task.status = TaskStatus.FAILED
                    self.task_queue.update_task_status(task.id, task.status)

def main():
    task_queue = TaskQueue()
    worker_thread = WorkerThread(task_queue)

    # Add tasks to the task queue
    task1 = Task(1, 1, lambda: print("Task 1 executed"))
    task2 = Task(2, 2, lambda: print("Task 2 executed"))

    task_queue.add_task(task2)

3. **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
4. **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).

#### Task Retry Mechanism

#### Graceful Shutdown

The system will include a mechanism for graceful shutdown. When the system is shut down, all worker threads will be stopped, and any pending tasks will be cancelled.
