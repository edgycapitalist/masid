### Overview

The rate limiter module is designed to provide a flexible and efficient way to limit the rate at which requests are made to a system. The module uses the token bucket algorithm to manage the rate limiting, and supports multiple named rate limiters, configurable rates, and burst sizes.

### Module Components

*   **RateLimiter**: This is the core class of the module, responsible for managing the token bucket and enforcing the rate limiting.
*   **RateLimiterDecorator**: This is a decorator that can be used to easily apply rate limiting to functions.
*   **RateLimiterManager**: This class is responsible for managing multiple named rate limiters.

### Token Bucket Algorithm

The token bucket algorithm is used to manage the rate limiting. The algorithm works as follows:

*   Tokens are added to the bucket at a constant rate (defined by the `rate` parameter).
*   Each time a request is made, a token is removed from the bucket.
*   If the bucket is empty, the request is blocked until a token is added to the bucket.
*   The `burst_size` parameter defines the maximum number of tokens that can be in the bucket at any time.

### Configurable Rate and Burst Size

The `rate` and `burst_size` parameters can be configured for each rate limiter. The `rate` parameter defines the number of requests per second that are allowed, and the `burst_size` parameter defines the maximum number of requests that can be made in a short period of time.

### Multiple Named Rate Limiters

Multiple named rate limiters can be created using the `RateLimiterManager` class. Each rate limiter can have its own `rate` and `burst_size` parameters.

### Thread-Safe Operation

The rate limiter module is designed to be thread-safe. The `RateLimiter` class uses a lock to ensure that only one thread can access the token bucket at a time.

### Example Use Cases

*   **Per-User Rate Limiter**: Create a rate limiter for each user, with a configurable `rate` and `burst_size` for each user.
*   **Per-Endpoint Rate Limiter**: Create a rate limiter for each endpoint, with a configurable `rate` and `burst_size` for each endpoint.

import threading
import time
from functools import wraps

class RateLimiter:
    def __init__(self, rate, burst_size):
        """
        Initialize the rate limiter.

        :param rate: The number of requests per second allowed.
        :param burst_size: The maximum number of requests that can be made in a short period of time.
        """
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        """
        Check if a request is allowed.

        :return: True if the request is allowed, False otherwise.
        """
        with self.lock:
            now = time.time()
            elapsed = now - self.last_update
            self.last_update = now
            self.tokens = min(self.burst_size, self.tokens + elapsed * self.rate)
            if self.tokens < 1:
                return False
            self.tokens -= 1
            return True

    def __call__(self, func):
        """
        Decorator to apply rate limiting to a function.

        :param func: The function to decorate.
        :return: The decorated function.
        """
        @wraps(func)
        def wrapper(*args, **kwargs):
            if not self.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper

class RateLimiterManager:
    def __init__(self):
        """
        Initialize the rate limiter manager.
        """
        self.rate_limiters = {}

    def get_rate_limiter(self, name, rate, burst_size):
        """
        Get a rate limiter by name.

        :param name: The name of the rate limiter.
        :param rate: The number of requests per second allowed.
        :param burst_size: The maximum number of requests that can be made in a short period of time.
        :return: The rate limiter.
        """
        if name not in self.rate_limiters:
            self.rate_limiters[name] = RateLimiter(rate, burst_size)
        return self.rate_limiters[name]

### Overview

The rate limiter module is designed to provide a flexible and efficient way to limit the rate at which requests are made to a system. The module uses the token bucket algorithm to manage the rate limiting, and supports multiple named rate limiters, configurable rates, and burst sizes.

### Module Components

*   **RateLimiter**: This is the core class of the module, responsible for managing the token bucket and enforcing the rate limiting.
*   **RateLimiterDecorator**: This is a decorator that can be used to easily apply rate limiting to functions.
*   **RateLimiterManager**: This class is responsible for managing multiple named rate limiters.

### Token Bucket Algorithm

The token bucket algorithm is used to manage the rate limiting. The algorithm works as follows:

*   Tokens are added to the bucket at a constant rate (defined by the `rate` parameter).
*   Each time a request is made, a token is removed from the bucket.
*   If the bucket is empty, the request is blocked until a token is added to the bucket.
*   The `burst_size` parameter defines the maximum number of tokens that can be in the bucket at any time.

### Configurable Rate and Burst Size

The `rate` and `burst_size` parameters can be configured for each rate limiter. The `rate` parameter defines the number of requests per second that are allowed, and the `burst_size` parameter defines the maximum number of requests that can be made in a short period of time.

### Multiple Named Rate Limiters

Multiple named rate limiters can be created using the `RateLimiterManager` class. Each rate limiter can have its own `rate` and `burst_size` parameters.

### Thread-Safe Operation

The rate limiter module is designed to be thread-safe. The `RateLimiter` class uses a lock to ensure that only one thread can access the token bucket at a time.

### Example Use Cases

*   **Per-User Rate Limiter**: Create a rate limiter for each user, with a configurable `rate` and `burst_size` for each user.
*   **Per-Endpoint Rate Limiter**: Create a rate limiter for each endpoint, with a configurable `rate` and `burst_size` for each endpoint.

### Edge Case Handling

The following edge cases are handled:

*   **Zero Rate**: If the rate is set to zero, the rate limiter will not allow any requests.
*   **Negative Rate**: If the rate is set to a negative value, the rate limiter will raise a `ValueError`.
*   **Zero Burst Size**: If the burst size is set to zero, the rate limiter will not allow any requests.
*   **Negative Burst Size**: If the burst size is set to a negative value, the rate limiter will raise a `ValueError`.
*   **Concurrent Requests**: If multiple requests are made concurrently, the rate limiter will ensure that only the allowed number of requests are made.

import threading
import time
from functools import wraps

class RateLimiter:
    def __init__(self, rate, burst_size):
        """
        Initialize the rate limiter.

        :param rate: The number of requests per second allowed.
        :param burst_size: The maximum number of requests that can be made in a short period of time.
        """
        if rate < 0:
            raise ValueError("Rate cannot be negative")
        if burst_size < 0:
            raise ValueError("Burst size cannot be negative")
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        """
        Check if a request is allowed.

        :return: True if the request is allowed, False otherwise.
        """
        with self.lock:
            now = time.time()
            elapsed = now - self.last_update
            self.last_update = now
            self.tokens = min(self.burst_size, self.tokens + elapsed * self.rate)
            if self.tokens < 1:
                return False
            self.tokens -= 1
            return True

    def __call__(self, func):
        """
        Decorator to apply rate limiting to a function.

        :param func: The function to decorate.
        :return: The decorated function.
        """
        @wraps(func)
        def wrapper(*args, **kwargs):
            if not self.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper

    def __init__(self):
        """
        Initialize the rate limiter manager.
        """
        self.rate_limiters = {}

        :param rate: The number of requests per second allowed.
        :param burst_size: The maximum number of requests that can be made in a short period of time.
        :return: The rate limiter.
        """

            self.rate_limiters[name] = RateLimiter(rate, burst_size)
        return self.rate_limiters[name]
