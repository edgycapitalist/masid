### Overview

The in-memory task queue is designed to manage tasks with priorities, allow worker threads to consume tasks, track task status, and support task retry with configurable max retries. The system will also provide a mechanism for graceful shutdown.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and a callable function to execute.
2. **Task Queue**: Manages tasks, allows adding tasks with priorities, and provides a mechanism for worker threads to consume tasks.
3. **Worker Thread**: Consumes tasks from the task queue, executes tasks, and updates task status.
4. **Task Status Tracker**: Tracks the status of tasks (pending, running, completed, failed).
5. **Retry Mechanism**: Supports task retry with configurable max retries.

### System Design

The system will use a priority queue to store tasks, allowing tasks with higher priorities to be executed first. The task queue will be implemented using a thread-safe data structure to ensure concurrent access.

The worker threads will consume tasks from the task queue, execute tasks, and update task status. The task status tracker will maintain a record of task status, and the retry mechanism will retry tasks that fail until the max retries limit is reached.

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task: Task, priority: int)`: Adds a task to the task queue with a specified priority.
	* `get_task()`: Retrieves the next task from the task queue based on priority.
2. **Worker Thread Interface**:
	* `start()`: Starts the worker thread.
	* `stop()`: Stops the worker thread.
3. **Task Status Tracker Interface**:
	* `get_task_status(task_id: str)`: Retrieves the status of a task.
	* `update_task_status(task_id: str, status: str)`: Updates the status of a task.

### Data Flow

1. **Task Addition**:
	* The task queue receives a task addition request with a task and priority.
	* The task queue adds the task to the priority queue.
2. **Task Consumption**:
	* A worker thread requests the next task from the task queue.
	* The task queue retrieves the next task based on priority and returns it to the worker thread.
3. **Task Execution**:
	* The worker thread executes the task and updates the task status.
	* The task status tracker maintains a record of task status.
4. **Task Retry**:
	* If a task fails, the retry mechanism retries the task until the max retries limit is reached.

import queue
import threading
from typing import Callable, Dict

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable):
        self.task_id = task_id
        self.priority = priority
        self.func = func

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            self.queue.put((task.priority, task))

    def get_task(self) -> Task:
        with self.lock:
            return self.queue.get()[1]

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue, max_retries: int):
        super().__init__()
        self.task_queue = task_queue
        self.max_retries = max_retries
        self.running = True

    def start(self):
        super().start()

    def stop(self):
        self.running = False

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            try:
                task.func()
                task_status = "completed"
            except Exception as e:
                task_status = "failed"
                if task.priority < self.max_retries:
                    self.task_queue.add_task(task)
            finally:
                self.task_queue.update_task_status(task.task_id, task_status)

class TaskStatusTracker:
    def __init__(self):

        return self.status.get(task_id)

    def update_task_status(self, task_id: str, status: str):
        self.status[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True
        self.max_retries = 3

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                retries = 0
                while retries < self.max_retries:
                    try:
                        task.func()
                        break
                    except Exception as e:
                        retries += 1
                        print(f"Task {task.task_id} failed: {str(e)}")
                else:
                    print(f"Task {task.task_id} failed after {self.max_retries} retries")
            else:
                time.sleep(0.1)

    def stop(self):
        self.running = False
