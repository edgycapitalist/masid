**Architecture Document: In-Memory Task Queue**
=====================================================

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks, track task status, and support task retry with configurable max retries. The system will also provide a mechanism for graceful shutdown.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
2. **Task Queue**: A data structure that stores tasks in a priority order, allowing worker threads to consume tasks.
3. **Worker Thread**: Responsible for consuming tasks from the task queue, executing tasks, and updating task status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Handles task retry with configurable max retries.

### System Flow

1. **Task Submission**:
	* A task is added to the task queue with a specified priority.
	* The task is assigned a unique ID and its status is set to "pending".
2. **Worker Thread Execution**:
	* A worker thread consumes a task from the task queue based on priority.
	* The task's status is updated to "running".
	* The worker thread executes the task's callable function.
	* If the task completes successfully, its status is updated to "completed".
	* If the task fails, its status is updated to "failed" and the retry mechanism is triggered.
3. **Retry Mechanism**:
	* If a task fails, it is added back to the task queue with a decreased priority.
	* The task's status is updated to "pending".
	* The retry mechanism will retry the task up to a configurable max retries.
4. **Graceful Shutdown**:
	* When the system is shut down, all worker threads are stopped.
	* Any pending tasks are preserved in the task queue.

### Implementation Details

* The task queue will be implemented using a priority queue data structure (e.g., `heapq` in Python).
* Worker threads will be implemented using the `threading` module in Python.
* Task status tracking will be implemented using a dictionary mapping task IDs to their respective statuses.
* The retry mechanism will be implemented using a counter to track the number of retries for each task.

import heapq
import threading
from typing import Callable, Dict

class Task:
    def __init__(self, task_id: int, priority: int, func: Callable, max_retries: int = 3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = "pending"

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.task_status: Dict[int, str] = {}

    def add_task(self, task: Task):
        heapq.heappush(self.queue, (task.priority, task))
        self.task_status[task.task_id] = "pending"

    def get_task(self) -> Task:
        if self.queue:
            return heapq.heappop(self.queue)[1]
        return None

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            task.status = "running"
            try:
                task.func()
                task.status = "completed"
            except Exception as e:
                task.status = "failed"
                if task.retries < task.max_retries:
                    task.retries += 1
                    self.task_queue.add_task(task)
                else:
                    print(f"Task {task.task_id} failed after {task.max_retries} retries")

def main():
    task_queue = TaskQueue()
    worker_threads = []

    # Add tasks to the task queue
    task1 = Task(1, 1, lambda: print("Task 1 executed"))
    task2 = Task(2, 2, lambda: print("Task 2 executed"))
    task_queue.add_task(task1)
    task_queue.add_task(task2)

    # Create and start worker threads
    for _ in range(2):
        worker_thread = WorkerThread(task_queue)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Wait for worker threads to finish
    for worker_thread in worker_threads:
        worker_thread.join()

    def __init__(self, db_file: str, max_size: int = 1000):
        self.queue = []
        self.task_status: Dict[int, str] = {}

        self.db_file = db_file
        self.conn = sqlite3.connect(db_file)

        self.cursor.execute("""

                task_id INTEGER PRIMARY KEY,
                priority INTEGER,
                func TEXT,
                max_retries INTEGER,
                retries INTEGER,
                status TEXT
            )

        self.max_size = max_size
