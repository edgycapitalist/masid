## Overview

The in-memory task queue is designed to support adding tasks with priorities, allowing worker threads to consume tasks, tracking task status, and supporting task retry with configurable max retries. The system will also include a mechanism for graceful shutdown.

## Components

*   **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
*   **Task Queue**: A data structure that stores tasks in memory, allowing worker threads to consume them.
*   **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
*   **Task Status Tracker**: Keeps track of the status of each task (pending, running, completed, failed).
*   **Retry Mechanism**: Allows tasks to be retried with a configurable maximum number of retries.

## System Design

The system will consist of the following components:

### Task Queue

*   The task queue will be implemented using a priority queue data structure (e.g., `heapq` in Python).
*   Tasks will be added to the queue with their corresponding priorities.
*   Worker threads will consume tasks from the queue based on their priority.

### Worker Thread

*   Worker threads will be implemented using Python's `threading` module.
*   Each worker thread will continuously consume tasks from the task queue and execute them.
*   If a task fails, the worker thread will retry the task with a configurable maximum number of retries.

### Task Status Tracker

*   The task status tracker will be implemented using a dictionary that maps task IDs to their corresponding status (pending, running, completed, failed).
*   The status of each task will be updated by the worker threads as they execute tasks.

### Retry Mechanism

*   The retry mechanism will be implemented using a counter that tracks the number of retries for each task.
*   If a task fails and the maximum number of retries has not been reached, the worker thread will retry the task.

```python
class TaskQueue:

    def add_task(self, task: Task):
        heapq.heappush(self.queue, (task.priority, task))
```

### Worker Thread Class

```python
import threading
from typing import Callable

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                try:
                    self.task_queue.update_task_status(task.id, "running")
                    task.func()
                    self.task_queue.update_task_status(task.id, "completed")
                except Exception as e:
                    self.task_queue.update_task_status(task.id, "failed")
                    if task.retries < task.max_retries:
                        task.retries += 1
                        self.task_queue.add_task(task)
                    else:
                        print(f"Task {task.id} failed after {task.max_retries} retries")
            else:
                # No tasks in the queue, sleep for a short period
                threading.Event().wait(0.1)
```

```python
        self.queue = []
        self.task_status = {}
        self.max_size = max_size
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            if len(self.queue) < self.max_size:
                self.queue.append(task)
                self.queue.sort(key=lambda x: x.id)
                self.task_status[task.id] = "pending"
            else:
                raise Exception("Task queue is full")

    def get_task(self):
        with self.lock:
            if self.queue:

            if task_id in self.task_status:
                self.task_status[task_id] = status
```
