## Overview

The in-memory task queue is designed to manage tasks with priorities, track their status, and provide retry capabilities. The system consists of the following components:

*   **Task**: Represents a unit of work with a unique identifier, priority, and status.
*   **Task Queue**: A data structure that stores tasks in memory, allowing for efficient insertion and retrieval based on priority.
*   **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
*   **Task Manager**: Responsible for adding tasks to the queue, tracking their status, and handling retries.

## System Design

### Task Queue

The task queue is implemented using a priority queue data structure, which ensures that tasks with higher priorities are executed first. The queue is stored in memory for optimal performance.

### Task Status Tracking

Tasks can have one of the following statuses:

*   **Pending**: Task is waiting in the queue to be executed.
*   **Running**: Task is being executed by a worker thread.
*   **Completed**: Task has been successfully executed.
*   **Failed**: Task has failed execution and may be retried.

### Task Retry

Tasks can be configured with a maximum number of retries. If a task fails, it will be retried up to the specified number of times before being marked as failed.

### Worker Threads

Worker threads consume tasks from the task queue, execute them, and update their status. Each worker thread runs in a separate thread to ensure concurrent execution of tasks.

### Task Manager

The task manager is responsible for adding tasks to the queue, tracking their status, and handling retries. It provides a simple API for adding tasks and retrieving their status.

## Implementation Details

The implementation is written in Python, utilizing the `threading` module for worker threads and the `queue` module for the priority queue. The `enum` module is used to define task statuses.

### Worker Thread Class

```python
import threading
from typing import Callable

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task is None:
                continue
            task.status = TaskStatus.RUNNING
            try:
                task.func()
                task.status = TaskStatus.COMPLETED
            except Exception as e:
                task.status = TaskStatus.FAILED
                task.retries += 1
                if task.retries < task.max_retries:
                    self.task_queue.add_task(task)
```

### Task Manager Class

```python
class TaskManager:
    def __init__(self):
        self.task_queue = TaskQueue()
        self.worker_threads = []

    def add_task(self, task: Task):
        self.task_queue.add_task(task)

    def start_workers(self, num_workers: int):
        for _ in range(num_workers):
            worker = WorkerThread(self.task_queue)
            worker.start()
            self.worker_threads.append(worker)

    def shutdown(self):
        for worker in self.worker_threads:
            worker.running = False
        for worker in self.worker_threads:
            worker.join()
```

    def __str__(self):

    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True
        self.paused = False
        self.notify_event = threading.Event()

    def run(self):
        while self.running:
            if self.paused:
                self.notify_event.wait()

                task.retries += 1
                if task.retries < task.max_retries:
                    self.task_queue.add_task(task)

        self.paused = False

## Overview

The in-memory task queue is designed to manage tasks with priorities, track their status, and provide retry capabilities. The system consists of the following components:

*   **Task**: Represents a unit of work with a unique identifier, priority, and status.
*   **Task Queue**: A data structure that stores tasks in memory, allowing for efficient insertion and retrieval based on priority.
*   **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
*   **Task Manager**: Responsible for adding tasks to the queue, tracking their status, and handling retries.

## System Design

### Task Queue

The task queue is implemented using a priority queue data structure, which ensures that tasks with higher priorities are executed first. The queue is stored in memory for optimal performance.

### Task Status Tracking

Tasks can have one of the following statuses:

*   **Pending**: Task is waiting in the queue to be executed.
*   **Running**: Task is being executed by a worker thread.
*   **Completed**: Task has been successfully executed.
*   **Failed**: Task has failed execution and may be retried.

### Task Retry

Tasks can be configured with a maximum number of retries. If a task fails, it will be retried up to the specified number of times before being marked as failed.

### Worker Threads

Worker threads consume tasks from the task queue, execute them, and update their status. Each worker thread runs in a separate thread to ensure concurrent execution of tasks.

### Task Manager

The task manager is responsible for adding tasks to the queue, tracking their status, and handling retries. It provides a simple API for adding tasks and retrieving their status.

## Implementation Details

The implementation is written in Python, utilizing the `threading` module for worker threads and the `queue` module for the priority queue. The `enum` module is used to define task statuses.
