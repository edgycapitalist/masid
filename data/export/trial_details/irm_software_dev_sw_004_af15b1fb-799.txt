5. **Graceful Shutdown**: The system shuts down worker threads and waits for any running tasks to complete.

from concurrent.futures import ThreadPoolExecutor

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id: int, priority: int, callback: Callable):

        self.priority = priority
        self.callback = callback
        self.status = TaskStatus.PENDING
        self.retry_count = 0

    def __init__(self):

        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            self.tasks.put((task.priority, task))

    def get_task(self) -> Task:
        with self.lock:
            if not self.tasks.empty():
                return self.tasks.get()[1]
            return None

        self.max_retries = max_retries

            self.tasks.put((task.priority, task))

    def get_task(self) -> Task:
        with self.lock:
            if not self.tasks.empty():
                return self.tasks.get()[1]
            return None

    def __init__(self, task_queue: TaskQueue, max_retries: int):
        self.task_queue = task_queue
        self.max_retries = max_retries

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break

            task.status = TaskStatus.RUNNING
            logger.info(f"Task {task.id} is running")
            try:
                task.callback()
                task.status = TaskStatus.COMPLETED
                logger.info(f"Task {task.id} completed")
            except Exception as e:
                task.status = TaskStatus.FAILED
                logger.error(f"Task {task.id} failed: {e}")
                if task.retry_count < self.max_retries:
                    task.retry_count += 1
                    self.task_queue.add_task(task)
                    logger.info(f"Task {task.id} will be retried")
                else:
                    logger.error(f"Task {task.id} failed after {self.max_retries} retries")

#### Introduction

The in-memory task queue is designed to manage tasks with priorities, allow worker threads to consume tasks, track task status, and support task retry with configurable max retries. The system will also provide a mechanism for graceful shutdown.

#### System Components

1.  **Task Queue**: A data structure that stores tasks with their corresponding priorities.
2.  **Task**: A unit of work with a unique ID, priority, and a callback function to execute.
3.  **Worker Thread**: A thread that consumes tasks from the task queue and executes them.
4.  **Task Status Tracker**: A component that tracks the status of each task (pending, running, completed, failed).
5.  **Retry Mechanism**: A component that retries failed tasks with a configurable max retries.

#### System Workflow

1.  **Task Addition**: A task is added to the task queue with a specified priority.
2.  **Task Consumption**: A worker thread consumes a task from the task queue and executes it.
3.  **Task Status Update**: The task status tracker updates the status of the task (running, completed, failed).
4.  **Task Retry**: If a task fails, the retry mechanism retries the task with a decrementing retry count.
5.  **Graceful Shutdown**: The system shuts down worker threads and waits for any running tasks to complete.

#### System Design

The system will utilize a combination of the following design patterns:

*   **Producer-Consumer Pattern**: The task queue will act as a producer, and the worker threads will act as consumers.
*   **Observer Pattern**: The task status tracker will observe the status of each task and update the task queue accordingly.
*   **Strategy Pattern**: The retry mechanism will use a strategy pattern to determine the retry policy.

#### Technology Choices

*   **Programming Language**: Python 3.9+
*   **Data Structure**: A priority queue (implemented using the `queue` module) will be used to store tasks.
*   **Threading Library**: The `concurrent.futures` module will be used to create worker threads.
