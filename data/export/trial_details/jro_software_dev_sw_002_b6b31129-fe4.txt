import threading
import time
from functools import wraps

class TokenBucket:
    def __init__(self, rate, burst_size):
        """
        Initializes the token bucket with the given rate and burst size.

        Args:
            rate (float): The rate at which tokens are added to the bucket.
            burst_size (int): The maximum number of tokens that can be in the bucket.
        """
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()

    def add_tokens(self):
        """
        Adds tokens to the bucket based on the time elapsed since the last update.
        """
        now = time.time()
        elapsed = now - self.last_update
        self.tokens = min(self.burst_size, self.tokens + elapsed * self.rate)
        self.last_update = now

    def consume_tokens(self, tokens):
        """
        Consumes the given number of tokens from the bucket.

        Args:
            tokens (int): The number of tokens to consume.

        Returns:
            bool: True if the tokens were consumed successfully, False otherwise.
        """
        self.add_tokens()
        if self.tokens >= tokens:
            self.tokens -= tokens
            return True
        return False

    def __init__(self, rate, burst_size, name):
        """
        Initializes the rate limiter with the given rate and burst size.

        Args:
            rate (float): The rate at which requests are allowed.

### Overview

The rate limiter module is designed to provide a flexible and thread-safe way to limit the rate at which requests are made to a system. The module uses the token bucket algorithm to manage the rate limiting, allowing for a configurable rate and burst size.

### Module Structure

The rate limiter module will consist of the following components:

*   `RateLimiter`: The main class that manages the rate limiting. It will have methods to check if a request is allowed, and to consume a token from the bucket.
*   `TokenBucket`: A class that implements the token bucket algorithm. It will have methods to add tokens to the bucket and to consume tokens from the bucket.
*   `RateLimiterDecorator`: A decorator class that can be used to easily apply rate limiting to functions.
*   `RateLimiterManager`: A class that manages multiple rate limiters, providing a way to create, delete, and update rate limiters.

class TokenBucket:
    def __init__(self, rate, burst_size):
        """
        Initializes the token bucket with the given rate and burst size.

        Args:
            rate (float): The rate at which tokens are added to the bucket.
            burst_size (int): The maximum number of tokens that can be in the bucket.
        """
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()

    def add_tokens(self):
        """
        Adds tokens to the bucket based on the time elapsed since the last update.
        """
        now = time.time()
        elapsed = now - self.last_update
        self.tokens = min(self.burst_size, self.tokens + elapsed * self.rate)
        self.last_update = now

    def consume_tokens(self, tokens):
        """
        Consumes the given number of tokens from the bucket.

        Args:
            tokens (int): The number of tokens to consume.

        Returns:
            bool: True if the tokens were consumed successfully, False otherwise.
        """
        self.add_tokens()
        if self.tokens >= tokens:
            self.tokens -= tokens
            return True
        return False
