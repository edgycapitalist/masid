### Overview

The URL shortening library is designed to provide a hash-based URL shortening functionality without relying on external services. The library will accept a long URL, generate a shortened version, and support reverse lookup from short URL to original URL. It will also handle collision detection, input validation, and ensure thread safety for concurrent usage.

### Module Decomposition

The library will consist of the following modules:

*   `url_shortener.py`: This module will contain the main URL shortening functionality.
*   `hash_generator.py`: This module will be responsible for generating hashes for the URLs.
*   `collision_detector.py`: This module will handle collision detection and resolution.
*   `input_validator.py`: This module will validate the input URLs to ensure they are in a valid format.
*   `thread_safe_storage.py`: This module will provide thread-safe storage for the URL mappings.

### Interfaces

The library will provide the following interfaces:

*   `shorten_url(long_url: str) -> str`: This function will take a long URL as input and return a shortened version.
*   `lookup_short_url(short_url: str) -> str`: This function will take a short URL as input and return the original URL.
*   `validate_url(url: str) -> bool`: This function will validate the input URL to ensure it is in a valid format.

### Data Flow

The data flow of the library will be as follows:

1.  The user will call the `shorten_url` function with a long URL as input.
2.  The `input_validator` module will validate the input URL to ensure it is in a valid format.
3.  If the input URL is valid, the `hash_generator` module will generate a hash for the URL.
4.  The `collision_detector` module will check for collisions and resolve them if necessary.
5.  The `thread_safe_storage` module will store the mapping between the short URL and the original URL.
6.  The `shorten_url` function will return the shortened URL to the user.

### Technology Choices

The library will be implemented in Python, and the following technologies will be used:

*   `hashlib` library for generating hashes
*   `threading` library for ensuring thread safety
*   `urllib.parse` library for parsing and validating URLs

import threading

class ThreadSafeStorage:
    def __init__(self):
        self.storage = {}
        self.lock = threading.Lock()

    def store_mapping(self, short_url: str, long_url: str):
        with self.lock:
            self.storage[short_url] = long_url

    def get_long_url(self, short_url: str) -> str:
        with self.lock:
            return self.storage.get(short_url)

    def get_short_url(self, hash_value: str) -> str:
        with self.lock:
            for short_url, long_url in self.storage.items():
                if generate_hash(long_url) == hash_value:
                    return short_url
            return None
