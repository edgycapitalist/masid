4. **Retry Mechanism**: A component that retries failed tasks with a configurable max retries.

	+ `args`: Task function arguments.

	+ `max_retries`: Maximum number of retries for the task.

### Task Status Tracker

* **Status Class**: A class that represents a task status with attributes:
	+ `id`: Task ID.

	+ `retries`: Number of retries for the task.

* **Retry Class**: A class that represents a retry attempt with attributes:
	+ `task_id`: Task ID.
	+ `retry_count`: Number of retries for the task.

## System Interactions

1. **Task Addition**: A task is added to the task queue with its respective priority.
2. **Task Consumption**: A worker thread consumes a task from the task queue and executes it.
3. **Task Status Update**: The task status tracker updates the task status based on the task execution result.
4. **Task Retry**: The retry mechanism retries a failed task with a configurable max retries.

import heapq
import threading
from enum import Enum

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, func, args, kwargs, max_retries):
        self.id = id
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.max_retries = max_retries

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self):
        with self.lock:
            if self.queue:
                return heapq.heappop(self.queue)[1]
            return None

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            self.task_status_tracker.update_status(task.id, TaskStatus.RUNNING)
            try:
                task.func(*task.args, **task.kwargs)
                self.task_status_tracker.update_status(task.id, TaskStatus.COMPLETED)
            except Exception as e:
                self.task_status_tracker.update_status(task.id, TaskStatus.FAILED)
                if task.max_retries > 0:
                    task.max_retries -= 1
                    self.task_queue.add_task(task)

class TaskStatusTracker:
    def __init__(self):
        self.statuses = {}

    def update_status(self, task_id, status):
        self.statuses[task_id] = status

class RetryMechanism:
    def __init__(self, task_queue, task_status_tracker):
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker

    def retry_task(self, task):
        if task.max_retries > 0:
            task.max_retries -= 1
            self.task_queue.add_task(task)

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()
    retry_mechanism = RetryMechanism(task_queue, task_status_tracker)

    # Add tasks to the task queue
    task1 = Task(1, 1, lambda: print("Task 1"), (), {}, 3)
    task2 = Task(2, 2, lambda: print("Task 2"), (), {}, 2)
    task_queue.add_task(task1)
    task_queue.add_task(task2)

    # Start worker threads

        worker_threads.append(worker_thread)

    # Wait for all tasks to complete

        self.args = args
        self.kwargs = kwargs
        self.max_retries = max_retries

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            try:
                heapq.heappush(self.queue, (task.priority, task))
                logger.info(f"Task {task.id} added to queue")
            except Exception as e:
                logger.error(f"Error adding task {task.id} to queue: {e}")

    def get_task(self):
        with self.lock:
            try:
                if self.queue:
                    return heapq.heappop(self.queue)[1]
                return None
            except Exception as e:
                logger.error(f"Error retrieving task from queue: {e}")

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker
        self.stop_event = threading.Event()

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.get_task()
            if task is None:
                continue
            try:
                self.task_status_tracker.update_status(task.id, TaskStatus.RUNNING)
                task.func(*task.args, **task.kwargs)
                self.task_status_tracker.update_status(task.id, TaskStatus.COMPLETED)
                logger.info(f"Task {task.id} completed successfully")
            except Exception as e:
                logger.error(f"Task {task.id} failed: {e}")
                self.task_status_tracker.update_status(task.id, TaskStatus.FAILED)
                if task.max_retries > 0:
                    task.max_retries -= 1
                    self.task_queue.add_task(task)

    def stop(self):
        self.stop_event.set()

## Introduction

The in-memory task queue is designed to support the execution of tasks with varying priorities, task status tracking, and task retry with configurable max retries. The system will consist of the following components:

1. **Task Queue**: A data structure that stores tasks with their respective priorities.
2. **Worker Threads**: Threads that consume tasks from the task queue.
3. **Task Status Tracker**: A component that tracks the status of each task (pending, running, completed, failed).
4. **Retry Mechanism**: A component that retries failed tasks with a configurable max retries.

## System Components

### Task Queue

* **Data Structure**: A priority queue (implemented using `heapq` in Python) to store tasks with their respective priorities.
* **Task Class**: A class that represents a task with attributes:
	+ `id`: Unique task ID.
	+ `priority`: Task priority.
	+ `func`: Task function to be executed.
	+ `args`: Task function arguments.
	+ `kwargs`: Task function keyword arguments.
	+ `max_retries`: Maximum number of retries for the task.

### Worker Threads

* **Thread Pool**: A pool of worker threads that consume tasks from the task queue.
* **Thread Class**: A class that represents a worker thread with attributes:
	+ `task_queue`: Reference to the task queue.
	+ `task_status_tracker`: Reference to the task status tracker.

### Task Status Tracker

* **Data Structure**: A dictionary that maps task IDs to their respective statuses.
* **Status Class**: A class that represents a task status with attributes:
	+ `id`: Task ID.
	+ `status`: Task status (pending, running, completed, failed).
	+ `retries`: Number of retries for the task.

### Retry Mechanism

* **Retry Policy**: A policy that determines when to retry a failed task.
* **Retry Class**: A class that represents a retry attempt with attributes:
	+ `task_id`: Task ID.
	+ `retry_count`: Number of retries for the task.

## System Interactions

1. **Task Addition**: A task is added to the task queue with its respective priority.
