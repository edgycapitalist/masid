### Overview

The rate limiter module is designed to implement the token bucket algorithm for rate limiting. It provides a flexible and thread-safe solution for limiting the rate of requests to a system. The module supports multiple named rate limiters, allowing for per-user, per-endpoint, or other custom rate limiting scenarios.

### Module Structure

The rate limiter module consists of the following components:

*   **RateLimiter**: The main class responsible for managing the rate limiting logic.
*   **TokenBucket**: A helper class that implements the token bucket algorithm.
*   **RateLimiterDecorator**: A decorator that simplifies the application of rate limiting to functions.

### Interfaces

The module exposes the following interfaces:

*   **RateLimiter**:
    *   `__init__(name: str, rate: float, burst_size: int)`: Initializes a new rate limiter with the given name, rate, and burst size.
    *   `allow() -> bool`: Checks if a request is allowed based on the current rate.
    *   `get_tokens() -> int`: Returns the current number of tokens in the bucket.
*   **RateLimiterDecorator**:
    *   `__init__(rate_limiter: RateLimiter)`: Initializes a new decorator with the given rate limiter.
    *   `__call__(func: Callable) -> Callable`: Applies the rate limiter to the given function.

### Data Flow

The data flow of the rate limiter module is as follows:

1.  The **RateLimiter** class initializes a new token bucket with the given rate and burst size.
2.  When a request is made, the **allow** method checks if there are sufficient tokens in the bucket to allow the request.
3.  If the request is allowed, the **allow** method consumes one token from the bucket and returns True.
4.  If the request is not allowed, the **allow** method returns False.
5.  The **RateLimiterDecorator** applies the rate limiter to a function by wrapping the function with the **allow** method.

    def __init__(self, rate: float, burst_size: int):
        """
        Initializes a new token bucket with the given rate and burst size.

        Args:
        rate (float): The rate at which tokens are added to the bucket per second.
        burst_size (int): The maximum number of tokens in the bucket.

        self.burst_size = burst_size
