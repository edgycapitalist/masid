**Architecture Document: In-Memory Task Queue**

## Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status and providing retry functionality. The system consists of the following components:

*   **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
*   **Task Queue**: Manages the tasks, providing methods to add tasks, consume tasks, and track task status.
*   **Worker Thread**: Consumes tasks from the task queue, executes the tasks, and updates the task status.
*   **Task Status**: Tracks the status of each task, which can be pending, running, completed, or failed.

## System Components

### Task

*   **Task ID**: Unique identifier for each task.
*   **Priority**: Priority of the task, with higher values indicating higher priority.
*   **Callable Function**: The function to be executed when the task is consumed.
*   **Max Retries**: The maximum number of times the task can be retried if it fails.

### Task Queue

*   **Task List**: A list of tasks, sorted by priority.
*   **Task Dictionary**: A dictionary mapping task IDs to their corresponding tasks.
*   **Lock**: A lock to ensure thread safety when accessing the task list and dictionary.

### Worker Thread

*   **Thread ID**: Unique identifier for each worker thread.
*   **Task Queue**: The task queue from which the worker thread consumes tasks.

## System Flow

1.  **Adding Tasks**:
    *   Create a new task with a unique ID, priority, callable function, and max retries.
    *   Add the task to the task queue, sorting the tasks by priority.
2.  **Consuming Tasks**:
    *   Worker threads consume tasks from the task queue, starting with the highest-priority task.
    *   When a task is consumed, its status is updated to "running."
3.  **Task Execution**:
    *   The worker thread executes the task's callable function.
    *   If the task fails, its status is updated to "failed," and it is retried if the max retries have not been exceeded.
4.  **Task Completion**:
    *   If the task is completed successfully, its status is updated to "completed."
5.  **Task Retry**:
    *   If a task fails and has not exceeded its max retries, it is retried by the worker thread.
6.  **Graceful Shutdown**:
    *   When the system is shut down, all worker threads are stopped, and any remaining tasks are marked as "failed."

import threading
from queue import PriorityQueue
from enum import Enum
import time

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id, priority, callable_function, max_retries):
        self.task_id = task_id
        self.priority = priority
        self.callable_function = callable_function
        self.max_retries = max_retries
        self.status = TaskStatus.PENDING
        self.retries = 0

    def execute(self):
        try:
            self.callable_function()
            self.status = TaskStatus.COMPLETED
        except Exception as e:
            self.status = TaskStatus.FAILED
            self.retries += 1
            if self.retries < self.max_retries:
                # Retry the task
                return False
        return True

class TaskQueue:
    def __init__(self):
        self.task_queue = PriorityQueue()
        self.task_dict = {}
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.task_queue.put((task.priority, task))
            self.task_dict[task.task_id] = task

    def consume_task(self):
        with self.lock:
            if self.task_queue.empty():
                return None
            _, task = self.task_queue.get()
            task.status = TaskStatus.RUNNING
            return task

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        threading.Thread.__init__(self)
        self.task_queue = task_queue
        self.stop_event = threading.Event()

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.consume_task()
            if task is None:
                time.sleep(0.1)  # Sleep for 100ms if no tasks are available
                continue
            if not task.execute():
                # Task failed and will be retried
                self.task_queue.add_task(task)

    def stop(self):
        self.stop_event.set()

# Example usage
def example_callable_function():
    print("Example callable function executed")

task_queue = TaskQueue()
worker_thread = WorkerThread(task_queue)

task = Task(1, 1, example_callable_function, 3)
task_queue.add_task(task)

worker_thread.start()

time.sleep(1)  # Let the worker thread execute the task
worker_thread.stop()
worker_thread.join()

## Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status and providing retry functionality. The system consists of the following components:

*   **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
*   **Task Queue**: Manages the tasks, providing methods to add tasks, consume tasks, and track task status.
*   **Worker Thread**: Consumes tasks from the task queue, executes the tasks, and updates the task status.
*   **Task Status**: Tracks the status of each task, which can be pending, running, completed, or failed.

## System Components

### Task

*   **Task ID**: Unique identifier for each task.
*   **Priority**: Priority of the task, with higher values indicating higher priority.
*   **Callable Function**: The function to be executed when the task is consumed.
*   **Max Retries**: The maximum number of times the task can be retried if it fails.

### Task Queue

*   **Task List**: A list of tasks, sorted by priority.
*   **Task Dictionary**: A dictionary mapping task IDs to their corresponding tasks.
*   **Lock**: A lock to ensure thread safety when accessing the task list and dictionary.

### Worker Thread

*   **Thread ID**: Unique identifier for each worker thread.
*   **Task Queue**: The task queue from which the worker thread consumes tasks.

## System Flow

1.  **Adding Tasks**:
    *   Create a new task with a unique ID, priority, callable function, and max retries.
    *   Add the task to the task queue, sorting the tasks by priority.
2.  **Consuming Tasks**:
    *   Worker threads consume tasks from the task queue, starting with the highest-priority task.
    *   When a task is consumed, its status is updated to "running."
3.  **Task Execution**:
    *   The worker thread executes the task's callable function.
    *   If the task fails, its status is updated to "failed," and it is retried if the max retries have not been exceeded.
4.  **Task Completion**:
    *   If the task is completed successfully, its status is updated to "completed."
5.  **Task Retry**:
    *   If a task fails and has not exceeded its max retries, it is retried by the worker thread.
6.  **Graceful Shutdown**:
    *   When the system is shut down, all worker threads are stopped, and any remaining tasks are marked as "failed."

import threading
from queue import PriorityQueue
from enum import Enum
import time
import logging

# Configure logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id, priority, callable_function, max_retries):
        self.task_id = task_id
        self.priority = priority
        self.callable_function = callable_function
        self.max_retries = max_retries
        self.status = TaskStatus.PENDING
        self.retries = 0

    def execute(self):
        try:
            self.callable_function()
            self.status = TaskStatus.COMPLETED
            logger.info(f"Task {self.task_id} completed successfully")
        except Exception as e:
            self.status = TaskStatus.FAILED
            self.retries += 1
            logger.error(f"Task {self.task_id} failed with error: {str(e)}")
            if self.retries < self.max_retries:
                # Retry the task
                return False
        return True

class TaskQueue:
    def __init__(self):
        self.task_queue = PriorityQueue()
        self.task_dict = {}
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.task_queue.put((task.priority, task))
            self.task_dict[task.task_id] = task
            logger.info(f"Task {task.task_id} added to the task queue")

    def consume_task(self):
        with self.lock:
            if self.task_queue.empty():
                return None
            _, task = self.task_queue.get()
            task.status = TaskStatus.RUNNING
            logger.info(f"Task {task.task_id} is running")
            return task

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        threading.Thread.__init__(self)
        self.task_queue = task_queue
        self.stop_event = threading.Event()

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.consume_task()
            if task is None:
                time.sleep(0.1)  # Sleep for 100ms if no tasks are available
                continue
            if not task.execute():
                # Task failed and will be retried
                self.task_queue.add_task(task)
                logger.info(f"Task {task.task_id} will be retried")

    def stop(self):
        self.stop_event.set()

class ThreadPool:
    def __init__(self, num_threads):
        self.num_threads = num_threads
        self.task_queue = TaskQueue()
        self.threads = []

    def start(self):
        for _ in range(self.num_threads):
            thread = WorkerThread(self.task_queue)
            thread.start()
            self.threads.append(thread)

    def stop(self):
        for thread in self.threads:
            thread.stop()
        for thread in self.threads:
            thread.join()

    def add_task(self, task):
        self.task_queue.add_task(task)
