**Architecture Document: In-Memory Task Queue**

## Overview
The in-memory task queue is designed to manage tasks with priorities, track their status, and provide a mechanism for worker threads to consume tasks. The system will also support task retry with a configurable maximum number of retries and ensure a graceful shutdown.

## Components
1. **Task**: Represents a unit of work with a priority, status, and a function to execute.
2. **Task Queue**: Holds tasks in memory, allowing for prioritized insertion and retrieval.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

## Design
### Task Queue
- Implemented as a priority queue using a binary heap data structure.
- Tasks are inserted based on their priority.
- The highest-priority task is always at the top of the heap.

### Worker Thread
- Consumes the highest-priority task from the task queue.
- Executes the task and updates its status accordingly.
- If a task fails, it is retried up to a configurable maximum number of times.

### Task Status Tracker
- Maintains a dictionary mapping task IDs to their respective statuses.
- Updates the status of a task as it progresses through its lifecycle.

### Graceful Shutdown
- When shutdown is initiated, no new tasks can be added to the queue.
- Existing worker threads are allowed to finish their current tasks.
- Once all tasks are completed or failed, the system shuts down.

import heapq
from enum import Enum
from typing import Callable, Dict

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable, max_retries: int = 3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.status_tracker: Dict[str, TaskStatus] = {}
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))
            self.status_tracker[task.task_id] = task.status

    def get_task(self) -> Task:
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

    def update_status(self, task_id: str, status: TaskStatus):
        with self.lock:
            self.status_tracker[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task is None:
                break
            try:
                self.task_queue.update_status(task.task_id, TaskStatus.RUNNING)
                task.func()
                self.task_queue.update_status(task.task_id, TaskStatus.COMPLETED)
            except Exception as e:
                if task.retries < task.max_retries:
                    task.retries += 1
                    self.task_queue.add_task(task)
                else:
                    self.task_queue.update_status(task.task_id, TaskStatus.FAILED)

    def stop(self):
        self.running = False

        self.lock = threading.Lock()

            if len(self.queue) < self.max_size:
                heapq.heappush(self.queue, (task.priority, task))
                self.status_tracker[task.task_id] = task.status
            else:

        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

        super().__init__()

        self.stop_event = stop_event

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.get_task()
            if task is None:
                continue

                self.task_queue.update_status(task.task_id, TaskStatus.RUNNING)

                    task.retries += 1

import threading
import heapq
from enum import Enum
from typing import Callable, Dict
import logging

# Set up logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable, max_retries: int = 3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self, max_size: int = 100):
        self.queue = []
        self.status_tracker: Dict[str, TaskStatus] = {}
        self.lock = threading.Lock()
        self.max_size = max_size

    def add_task(self, task: Task):
        with self.lock:
            if len(self.queue) < self.max_size:
                heapq.heappush(self.queue, (task.priority, task))
                self.status_tracker[task.task_id] = task.status
                logger.info(f"Task {task.task_id} added to queue")
            else:
                logger.warning("Task queue is full. Cannot add task.")

    def get_task(self) -> Task:
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

    def update_status(self, task_id: str, status: TaskStatus):
        with self.lock:
            self.status_tracker[task_id] = status
            logger.info(f"Task {task_id} status updated to {status}")

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue, stop_event: threading.Event):
        super().__init__()
        self.task_queue = task_queue
        self.stop_event = stop_event

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.get_task()
            if task is None:
                continue
            try:
                self.task_queue.update_status(task.task_id, TaskStatus.RUNNING)
                task.func()
                self.task_queue.update_status(task.task_id, TaskStatus.COMPLETED)
            except Exception as e:
                logger.error(f"Task {task.task_id} failed: {e}")
                if task.retries < task.max_retries:
                    task.retries += 1
                    self.task_queue.add_task(task)
                else:
                    self.task_queue.update_status(task.task_id, TaskStatus.FAILED)

**Revised Architecture Document: In-Memory Task Queue**

## Overview
The in-memory task queue is designed to manage tasks with priorities, track their status, and provide a mechanism for worker threads to consume tasks. The system will also support task retry with a configurable maximum number of retries and ensure a graceful shutdown.

## Components
1. **Task**: Represents a unit of work with a priority, status, and a function to execute.
2. **Task Queue**: Holds tasks in memory, allowing for prioritized insertion and retrieval.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

## Design
### Task Queue
- Implemented as a priority queue using a binary heap data structure.
- Tasks are inserted based on their priority.
- The highest-priority task is always at the top of the heap.

### Worker Thread
- Consumes the highest-priority task from the task queue.
- Executes the task and updates its status accordingly.
- If a task fails, it is retried up to a configurable maximum number of times.

### Task Status Tracker
- Maintains a dictionary mapping task IDs to their respective statuses.
- Updates the status of a task as it progresses through its lifecycle.

### Graceful Shutdown
- When shutdown is initiated, no new tasks can be added to the queue.
- Existing worker threads are allowed to finish their current tasks.
- Once all tasks are completed or failed, the system shuts down.
