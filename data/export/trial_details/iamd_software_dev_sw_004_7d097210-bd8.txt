## Introduction

The in-memory task queue is designed to manage tasks with priorities, track their status, and provide retry mechanisms. The system consists of the following components:

*   **Task**: Represents a unit of work with a priority, status, and retry count.
*   **Task Queue**: A data structure that stores tasks and provides methods for adding, consuming, and tracking tasks.
*   **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
*   **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

## System Components

### Task

*   **Attributes**:
    *   `id`: Unique identifier for the task.
    *   `priority`: Priority of the task (higher value means higher priority).
    *   `status`: Current status of the task (pending, running, completed, failed).
    *   `max_retries`: Maximum number of retries allowed for the task.
    *   `retry_count`: Current retry count for the task.
*   **Methods**:
    *   `execute()`: Executes the task.

### Task Queue

*   **Attributes**:
    *   `tasks`: A list of tasks in the queue.
*   **Methods**:
    *   `add_task(task)`: Adds a task to the queue.
    *   `consume_task()`: Removes and returns the next task from the queue based on priority.
    *   `update_task_status(task_id, status)`: Updates the status of a task.

### Worker Thread

*   **Attributes**:
    *   `task_queue`: Reference to the task queue.
*   **Methods**:
    *   `run()`: Consumes tasks from the task queue and executes them.

### Task Status Tracker

*   **Attributes**:
    *   `task_status`: A dictionary mapping task IDs to their current status.
*   **Methods**:
    *   `update_task_status(task_id, status)`: Updates the status of a task.
    *   `get_task_status(task_id)`: Returns the current status of a task.

## System Flow

1.  **Task Creation**: A task is created with a priority, max retries, and an execution function.
2.  **Task Addition**: The task is added to the task queue.
3.  **Worker Thread**: A worker thread consumes the next task from the task queue based on priority.
4.  **Task Execution**: The worker thread executes the task.
5.  **Task Status Update**: The task status is updated to "running".
6.  **Task Completion**: If the task completes successfully, its status is updated to "completed". If it fails, its status is updated to "failed" and it is retried if the retry count is less than the max retries.
7.  **Shutdown**: The system is shut down, and any remaining tasks are cancelled.

import threading
import time
from queue import PriorityQueue
from typing import Callable

class Task:
    def __init__(self, id: int, priority: int, max_retries: int, execute_func: Callable):
        self.id = id
        self.priority = priority
        self.max_retries = max_retries
        self.retry_count = 0
        self.status = "pending"
        self.execute_func = execute_func

    def execute(self):
        try:
            self.execute_func()
            self.status = "completed"
        except Exception as e:
            self.status = "failed"
            self.retry_count += 1
            if self.retry_count < self.max_retries:
                # Retry the task
                self.status = "pending"

class TaskQueue:
    def __init__(self):
        self.tasks = PriorityQueue()
        self.task_status = {}

    def add_task(self, task: Task):
        self.tasks.put((task.priority, task))
        self.task_status[task.id] = task.status

    def consume_task(self) -> Task:
        if self.tasks.empty():
            return None
        _, task = self.tasks.get()
        return task

    def update_task_status(self, task_id: int, status: str):
        if task_id in self.task_status:
            self.task_status[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.consume_task()
            if task is None:
                time.sleep(1)  # Wait for 1 second before checking again
                continue
            self.task_queue.update_task_status(task.id, "running")
            task.execute()
            self.task_queue.update_task_status(task.id, task.status)

    def stop(self):
        self.running = False

## Introduction

The in-memory task queue is designed to manage tasks with priorities, track their status, and provide retry mechanisms. The system consists of the following components:

*   **Task**: Represents a unit of work with a priority, status, and retry count.
*   **Task Queue**: A data structure that stores tasks and provides methods for adding, consuming, and tracking tasks.
*   **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
*   **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

## System Components

### Task

*   **Attributes**:
    *   `id`: Unique identifier for the task.
    *   `priority`: Priority of the task (higher value means higher priority).
    *   `status`: Current status of the task (pending, running, completed, failed).
    *   `max_retries`: Maximum number of retries allowed for the task.
    *   `retry_count`: Current retry count for the task.
*   **Methods**:
    *   `execute()`: Executes the task.

### Task Queue

*   **Attributes**:
    *   `tasks`: A list of tasks in the queue.
*   **Methods**:
    *   `add_task(task)`: Adds a task to the queue.
    *   `consume_task()`: Removes and returns the next task from the queue based on priority.
    *   `update_task_status(task_id, status)`: Updates the status of a task.
    *   `get_task_status(task_id)`: Returns the current status of a task.

### Worker Thread

*   **Attributes**:
    *   `task_queue`: Reference to the task queue.
*   **Methods**:
    *   `run()`: Consumes tasks from the task queue and executes them.
    *   `stop()`: Stops the worker thread.

### Task Status Tracker

*   **Attributes**:
    *   `task_status`: A dictionary mapping task IDs to their current status.
*   **Methods**:
    *   `update_task_status(task_id, status)`: Updates the status of a task.
    *   `get_task_status(task_id)`: Returns the current status of a task.

## System Flow

1.  **Task Creation**: A task is created with a priority, max retries, and an execution function.
2.  **Task Addition**: The task is added to the task queue.
3.  **Worker Thread**: A worker thread consumes the next task from the task queue based on priority.
4.  **Task Execution**: The worker thread executes the task.
5.  **Task Status Update**: The task status is updated to "running".
6.  **Task Completion**: If the task completes successfully, its status is updated to "completed". If it fails, its status is updated to "failed" and it is retried if the retry count is less than the max retries.
7.  **Shutdown**: The system is shut down, and any remaining tasks are cancelled.
