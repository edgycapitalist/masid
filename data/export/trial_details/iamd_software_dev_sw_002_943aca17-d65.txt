**Architecture Document: Rate Limiter Module**

## Introduction

The rate limiter module is designed to limit the rate at which requests are processed by an application. This is achieved using the token bucket algorithm, which allows for a configurable rate and burst size. The module supports multiple named rate limiters, making it suitable for per-user, per-endpoint, or other granular rate limiting requirements. The module is designed to be thread-safe and provides a decorator for easy application to functions.

## Module Structure

The rate limiter module consists of the following components:

*   `RateLimiter`: The core class that implements the token bucket algorithm and provides methods for checking and consuming tokens.
*   `NamedRateLimiter`: A dictionary-based wrapper around the `RateLimiter` class, allowing for multiple named rate limiters.
*   `rate_limited`: A decorator that applies the rate limiter to a function.

## Token Bucket Algorithm

The token bucket algorithm works as follows:

*   Tokens are added to a bucket at a constant rate (requests per second).
*   When a request is made, a token is consumed from the bucket.
*   If the bucket is empty, the request is blocked until a token becomes available.
*   The burst size determines the maximum number of tokens that can be accumulated in the bucket.

import threading
import time
from collections import defaultdict

class RateLimiter:
    def __init__(self, rate, burst_size):
        """
        Initialize the rate limiter.

        :param rate: The rate at which tokens are added to the bucket (requests per second).
        :param burst_size: The maximum number of tokens that can be accumulated in the bucket.
        """
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def consume(self, amount=1):
        """
        Consume tokens from the bucket.

        :param amount: The number of tokens to consume.
        :return: True if the tokens were consumed, False otherwise.
        """
        with self.lock:
            self._update_tokens()
            if self.tokens >= amount:
                self.tokens -= amount
                return True
            return False

    def _update_tokens(self):
        """
        Update the number of tokens in the bucket based on the elapsed time.
        """
        now = time.time()
        elapsed = now - self.last_update
        self.last_update = now
        self.tokens = min(self.burst_size, self.tokens + elapsed * self.rate)

class NamedRateLimiter:
    def __init__(self):
        self.limiters = defaultdict(dict)

    def get_limiter(self, name, rate, burst_size):
        """
        Get or create a rate limiter for the given name.

        :param name: The name of the rate limiter.
        :param rate: The rate at which tokens are added to the bucket (requests per second).
        :param burst_size: The maximum number of tokens that can be accumulated in the bucket.
        :return: The rate limiter for the given name.
        """
        if name not in self.limiters:
            self.limiters[name] = RateLimiter(rate, burst_size)
        return self.limiters[name]

def rate_limited(name, rate, burst_size):
    """
    Decorator to apply the rate limiter to a function.

    :param name: The name of the rate limiter.
    :param rate: The rate at which tokens are added to the bucket (requests per second).
    :param burst_size: The maximum number of tokens that can be accumulated in the bucket.
    :return: The decorated function.
    """
    limiter = NamedRateLimiter().get_limiter(name, rate, burst_size)

    def decorator(func):
        def wrapper(*args, **kwargs):
            if not limiter.consume():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

## Token Bucket Algorithm

The token bucket algorithm works as follows:

*   Tokens are added to a bucket at a constant rate (requests per second).

*   If the bucket is empty, the request is blocked until a token becomes available.
