**Architecture Document: In-Memory Task Queue**

## Overview
The in-memory task queue is designed to manage tasks with varying priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with a configurable maximum number of retries and provides a mechanism for graceful shutdown.

## Components

1. **Task**: Represents a single unit of work with a unique identifier, priority, and a callable function.
2. **Task Queue**: Manages the lifecycle of tasks, including adding, retrieving, and updating task status.
3. **Worker Thread**: Consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: Monitors the status of tasks, including pending, running, completed, and failed.

## System Design

### Task Queue

The task queue is implemented as a priority queue, where tasks are ordered based on their priority. The queue is divided into four sections:

* **Pending**: Tasks that are waiting to be executed.
* **Running**: Tasks that are currently being executed.
* **Completed**: Tasks that have completed successfully.
* **Failed**: Tasks that have failed and are awaiting retry or have exceeded the maximum number of retries.

### Worker Thread

Worker threads are responsible for consuming tasks from the pending section of the task queue. When a task is retrieved, it is moved to the running section. If the task completes successfully, it is moved to the completed section. If the task fails, it is moved to the failed section and retried if the maximum number of retries has not been exceeded.

### Task Status Tracker

The task status tracker monitors the status of tasks and provides a mechanism for updating task status. The tracker uses a dictionary to store task status, where each task is identified by a unique identifier.

### Task Retry

Tasks can be configured to retry a maximum number of times. If a task fails and the maximum number of retries has not been exceeded, the task is retried. If the task fails and the maximum number of retries has been exceeded, the task is moved to the failed section.

### Graceful Shutdown

The system provides a mechanism for graceful shutdown, where worker threads are stopped and tasks are prevented from being added to the queue.

import threading
import queue
import time
from enum import Enum
from functools import wraps

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, func, max_retries=3):
        self.id = id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.status_tracker = {}
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.queue.put((task.priority, task))
            self.status_tracker[task.id] = task.status

    def get_task(self):
        with self.lock:
            if self.queue.empty():
                return None
            _, task = self.queue.get()
            task.status = TaskStatus.RUNNING
            self.status_tracker[task.id] = task.status
            return task

    def update_task_status(self, task, status):
        with self.lock:
            task.status = status
            self.status_tracker[task.id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue
        self.stop_event = threading.Event()

    def run(self):
        while not self.stop_event.is_set():
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(0.1)
                continue
            try:
                task.func()
                self.task_queue.update_task_status(task, TaskStatus.COMPLETED)
            except Exception as e:
                task.retries += 1
                if task.retries < task.max_retries:
                    self.task_queue.update_task_status(task, TaskStatus.PENDING)
                else:
                    self.task_queue.update_task_status(task, TaskStatus.FAILED)
                    print(f"Task {task.id} failed with error: {str(e)}")

    def stop(self):
        self.stop_event.set()

def main():
    task_queue = TaskQueue()
    worker_thread = WorkerThread(task_queue)

    def example_task():

        self.lock = threading.Lock()

            self.queue.put((task.priority, task))
            self.status_tracker[task.id] = task.status
            logging.info(f"Task {task.id} added to queue")

            if self.queue.empty():

            _, task = self.queue.get()
            task.status = TaskStatus.RUNNING
            self.status_tracker[task.id] = task.status
