## Introduction
The in-memory task queue is designed to manage tasks with priorities, provide task status tracking, support task retry, and enable worker threads to consume tasks. The system consists of the following components:

1. **Task Queue**: Responsible for storing tasks with their respective priorities and statuses.
2. **Worker Threads**: Consume tasks from the task queue, execute them, and update their statuses.
3. **Task Manager**: Manages task statuses, handles task retries, and provides a shutdown mechanism.

## System Components

### Task Queue
The task queue is implemented using a priority queue data structure, where tasks are ordered based on their priorities. The task queue supports the following operations:

* `add_task(task)`: Adds a task to the queue with its priority and initial status (pending).
* `get_task()`: Retrieves the highest-priority task from the queue.

### Worker Threads
Worker threads are responsible for consuming tasks from the task queue, executing them, and updating their statuses. Each worker thread runs in an infinite loop, continuously fetching tasks from the queue and executing them until the shutdown signal is received.

### Task Manager
The task manager is responsible for managing task statuses, handling task retries, and providing a shutdown mechanism. It supports the following operations:

* `update_task_status(task_id, status)`: Updates the status of a task.
* `retry_task(task_id)`: Retries a failed task up to a configurable maximum number of retries.
* `shutdown()`: Initiates a graceful shutdown of the system.

## System Flow

1. A task is added to the task queue with its priority and initial status (pending).
2. A worker thread consumes the task from the queue and executes it.
3. The worker thread updates the task status to "running".
4. If the task execution fails, the worker thread updates the task status to "failed" and notifies the task manager.
5. The task manager retries the failed task up to a configurable maximum number of retries.
6. If all retries fail, the task status is updated to "failed".
7. If the task execution is successful, the worker thread updates the task status to "completed".
8. The system continues to run until the shutdown signal is received.

import threading
import queue
import time
import logging

class Task:
    def __init__(self, task_id, priority, func, *args, **kwargs):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.status = "pending"
        self.retries = 0
        self.max_retries = 3

    def execute(self):
        try:
            self.func(*self.args, **self.kwargs)
            self.status = "completed"
        except Exception as e:
            self.status = "failed"
            logging.error(f"Task {self.task_id} failed: {str(e)}")
            self.retries += 1
            if self.retries < self.max_retries:
                logging.info(f"Retrying task {self.task_id}")
                return False
        return True

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))

    def get_task(self):
        if self.queue.empty():
            return None
        return self.queue.get()[1]

class TaskManager:
    def __init__(self, task_queue):
        self.task_queue = task_queue
        self.shutdown = False
        self.lock = threading.Lock()

    def update_task_status(self, task_id, status):
        with self.lock:
            task = next((task for task in self.task_queue.queue), None)
            if task and task.task_id == task_id:
                task.status = status

    def retry_task(self, task_id):
        with self.lock:
            task = next((task for task in self.task_queue.queue), None)
            if task and task.task_id == task_id:
                if task.retries < task.max_retries:
                    task.retries += 1
                    logging.info(f"Retrying task {task_id}")
                    return True
        return False

        self.shutdown = True

        super().__init__()
        self.task_manager = task_manager

        while not self.task_manager.shutdown:
            task = self.task_manager.task_queue.get_task()
            if task:
                task.status = "running"
                if not task.execute():

                else:
