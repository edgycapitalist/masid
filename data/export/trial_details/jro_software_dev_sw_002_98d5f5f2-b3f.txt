**Rate Limiter Module Design Document**
======================================

**Introduction**
---------------

The rate limiter module is designed to support multiple named rate limiters using the token bucket algorithm. The module will provide a decorator for easy application to functions and ensure thread-safe operation.

**Module Structure**
-------------------

The rate limiter module will consist of the following components:

*   `RateLimiter`: The main class responsible for managing rate limiters.
*   `TokenBucket`: A class implementing the token bucket algorithm.
*   `RateLimiterDecorator`: A decorator for applying rate limiting to functions.

**Token Bucket Algorithm**
-------------------------

The token bucket algorithm is used to implement rate limiting. The algorithm works as follows:

*   A bucket is filled with tokens at a specified rate (requests per second).
*   Each request consumes one token from the bucket.
*   If the bucket is empty, the request is blocked until a token is available.

**Configurable Rate and Burst Size**
------------------------------------

The rate limiter module will allow for configurable rate (requests per second) and burst size. The burst size represents the maximum number of tokens that can be stored in the bucket.

**Multiple Named Rate Limiters**
------------------------------

The rate limiter module will support multiple named rate limiters. This allows for rate limiting to be applied per-user, per-endpoint, or any other criteria.

**Thread-Safe Operation**
----------------------

The rate limiter module will ensure thread-safe operation by using locks to synchronize access to the token bucket.

import threading
import time
from functools import wraps

class TokenBucket:
    """Token bucket implementation."""
    
    def __init__(self, rate, burst_size):
        """
        Initialize the token bucket.
        
        :param rate: The rate at which tokens are added to the bucket (requests per second).
        :param burst_size: The maximum number of tokens that can be stored in the bucket.
        """
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def consume(self, amount=1):
        """
        Consume tokens from the bucket.
        
        :param amount: The number of tokens to consume.
        :return: True if the tokens are available, False otherwise.
        """
        with self.lock:
            current_time = time.time()
            elapsed_time = current_time - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = current_time

            if self.tokens >= amount:
                self.tokens -= amount
                return True
            else:
                return False


class RateLimiter:
    """Rate limiter class."""
    
    def __init__(self):
        """Initialize the rate limiter."""
        self.rate_limiters = {}
        self.lock = threading.Lock()

    def add_rate_limiter(self, name, rate, burst_size):
        """
        Add a rate limiter.
        
        :param name: The name of the rate limiter.
        :param rate: The rate at which tokens are added to the bucket (requests per second).
        :param burst_size: The maximum number of tokens that can be stored in the bucket.
        """
        with self.lock:
            if name not in self.rate_limiters:
                self.rate_limiters[name] = TokenBucket(rate, burst_size)

    def get_rate_limiter(self, name):
        """
        Get a rate limiter by name.
        
        :param name: The name of the rate limiter.
        :return: The rate limiter if it exists, None otherwise.
        """
        with self.lock:
            return self.rate_limiters.get(name)


def rate_limited(rate_limiter_name, rate_limiter):
    """
    Decorator for applying rate limiting to functions.
    
    :param rate_limiter_name: The name of the rate limiter.
    :param rate_limiter: The rate limiter instance.
    """
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            rate_limiter_instance = rate_limiter.get_rate_limiter(rate_limiter_name)
            if rate_limiter_instance and not rate_limiter_instance.consume():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

**Rate Limiter Module Design Document**

**Introduction**
---------------

The rate limiter module is designed to support multiple named rate limiters using the token bucket algorithm. The module will provide a decorator for easy application to functions and ensure thread-safe operation.

**Module Structure**
-------------------

The rate limiter module will consist of the following components:

*   `RateLimiter`: The main class responsible for managing rate limiters.
*   `TokenBucket`: A class implementing the token bucket algorithm.
*   `RateLimiterDecorator`: A decorator for applying rate limiting to functions.

**Token Bucket Algorithm**
-------------------------

The token bucket algorithm is used to implement rate limiting. The algorithm works as follows:

*   A bucket is filled with tokens at a specified rate (requests per second).
*   Each request consumes one token from the bucket.
*   If the bucket is empty, the request is blocked until a token is available.

**Configurable Rate and Burst Size**
------------------------------------

The rate limiter module will allow for configurable rate (requests per second) and burst size. The burst size represents the maximum number of tokens that can be stored in the bucket.

**Multiple Named Rate Limiters**
------------------------------

The rate limiter module will support multiple named rate limiters. This allows for rate limiting to be applied per-user, per-endpoint, or any other criteria.

**Thread-Safe Operation**
----------------------

The rate limiter module will ensure thread-safe operation by using locks to synchronize access to the token bucket.

**Introduction**
---------------

The rate limiter module is designed to support multiple named rate limiters using the token bucket algorithm. The module will provide a decorator for easy application to functions and ensure thread-safe operation.

**Module Structure**
-------------------

The rate limiter module will consist of the following components:

*   `RateLimiter`: The main class responsible for managing rate limiters.
*   `TokenBucket`: A class implementing the token bucket algorithm.
*   `RateLimiterDecorator`: A decorator for applying rate limiting to functions.

**Token Bucket Algorithm**
-------------------------

The token bucket algorithm is used to implement rate limiting. The algorithm works as follows:

*   A bucket is filled with tokens at a specified rate (requests per second).
*   Each request consumes one token from the bucket.
*   If the bucket is empty, the request is blocked until a token is available.

**Configurable Rate and Burst Size**
------------------------------------

The rate limiter module will allow for configurable rate (requests per second) and burst size. The burst size represents the maximum number of tokens that can be stored in the bucket.

**Multiple Named Rate Limiters**
------------------------------

The rate limiter module will support multiple named rate limiters. This allows for rate limiting to be applied per-user, per-endpoint, or any other criteria.

**Thread-Safe Operation**
----------------------

The rate limiter module will ensure thread-safe operation by using locks to synchronize access to the token bucket.

class TokenBucket:
    """Token bucket implementation."""
    
    def __init__(self, rate, burst_size):
        """
        Initialize the token bucket.
        
        :param rate: The rate at which tokens are added to the bucket (requests per second).
        :param burst_size: The maximum number of tokens that can be stored in the bucket.
        """
        if rate < 0:
            raise ValueError("Rate must be non-negative")
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

        """
        Consume tokens from the bucket.
        
        :param amount: The number of tokens to consume.
        :return: True if the tokens are available, False otherwise.

        with self.lock:
            current_time = time.time()
            elapsed_time = current_time - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
