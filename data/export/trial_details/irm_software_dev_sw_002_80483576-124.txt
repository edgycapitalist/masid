## Introduction

The rate limiter module is designed to provide a flexible and efficient way to limit the rate at which requests are processed. The module implements the token bucket algorithm, which allows for a configurable rate and burst size. The module also supports multiple named rate limiters, making it suitable for a variety of use cases, such as per-user or per-endpoint rate limiting.

## Module Components

The rate limiter module consists of the following components:

1. **RateLimiter**: The main class responsible for implementing the token bucket algorithm.
2. **RateLimiterManager**: A class responsible for managing multiple named rate limiters.
3. **rate_limited**: A decorator that can be used to easily apply rate limiting to functions.

## Token Bucket Algorithm

The token bucket algorithm works as follows:

1. Tokens are added to the bucket at a constant rate (configurable).
2. Each request consumes a token from the bucket.
3. If the bucket is empty, the request is blocked until a token is available.
4. The bucket has a maximum size (configurable), which allows for bursts of requests.

## RateLimiter Class

The **RateLimiter** class has the following methods:

* `__init__(rate, burst_size)`: Initializes the rate limiter with the specified rate and burst size.
* `allow_request()`: Checks if a request is allowed based on the current state of the token bucket.
* `add_tokens(num_tokens)`: Adds tokens to the bucket.

## RateLimiterManager Class

The **RateLimiterManager** class has the following methods:

* `__init__()`: Initializes the rate limiter manager.
* `get_rate_limiter(name)`: Returns a rate limiter with the specified name.
* `add_rate_limiter(name, rate, burst_size)`: Adds a new rate limiter with the specified name, rate, and burst size.

## rate_limited Decorator

The **rate_limited** decorator takes a rate limiter name as an argument and applies rate limiting to the decorated function.

import threading
import time
from functools import wraps

class RateLimiter:
    def __init__(self, rate, burst_size):
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        with self.lock:
            current_time = time.time()
            elapsed_time = current_time - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = current_time
            if self.tokens < 1:
                return False
            self.tokens -= 1
            return True

    def add_tokens(self, num_tokens):
        with self.lock:
            self.tokens = min(self.burst_size, self.tokens + num_tokens)

class RateLimiterManager:
    def __init__(self):
        self.rate_limiters = {}
        self.lock = threading.Lock()

    def get_rate_limiter(self, name):
        with self.lock:
            if name not in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' not found")
            return self.rate_limiters[name]

    def add_rate_limiter(self, name, rate, burst_size):
        with self.lock:
            if name in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' already exists")
            self.rate_limiters[name] = RateLimiter(rate, burst_size)

def rate_limited(rate_limiter_name):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            rate_limiter_manager = RateLimiterManager()
            rate_limiter = rate_limiter_manager.get_rate_limiter(rate_limiter_name)
            if not rate_limiter.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

# Example usage:

rate_limiter_manager.add_rate_limiter("my_rate_limiter", 5, 10)

The rate limiter module is designed to provide a flexible and efficient way to limit the rate at which requests are processed. The module implements the token bucket algorithm, which allows for a configurable rate and burst size. The module also supports multiple named rate limiters, making it suitable for a variety of use cases, such as per-user or per-endpoint rate limiting.

## Module Components

The rate limiter module consists of the following components:

1. **RateLimiter**: The main class responsible for implementing the token bucket algorithm.
2. **RateLimiterManager**: A class responsible for managing multiple named rate limiters.
3. **rate_limited**: A decorator that can be used to easily apply rate limiting to functions.

## Token Bucket Algorithm

The token bucket algorithm works as follows:

1. Tokens are added to the bucket at a constant rate (configurable).
2. Each request consumes a token from the bucket.
3. If the bucket is empty, the request is blocked until a token is available.
4. The bucket has a maximum size (configurable), which allows for bursts of requests.

## RateLimiter Class

The **RateLimiter** class has the following methods:

* `__init__(rate, burst_size)`: Initializes the rate limiter with the specified rate and burst size.
* `allow_request()`: Checks if a request is allowed based on the current state of the token bucket.
* `add_tokens(num_tokens)`: Adds tokens to the bucket.
* `get_rate()`: Returns the current rate of the rate limiter.
* `get_burst_size()`: Returns the current burst size of the rate limiter.

## RateLimiterManager Class

The **RateLimiterManager** class has the following methods:

* `__init__()`: Initializes the rate limiter manager.
* `get_rate_limiter(name)`: Returns a rate limiter with the specified name.
* `add_rate_limiter(name, rate, burst_size)`: Adds a new rate limiter with the specified name, rate, and burst size.
* `remove_rate_limiter(name)`: Removes a rate limiter with the specified name.
* `update_rate_limiter(name, rate, burst_size)`: Updates a rate limiter with the specified name, rate, and burst size.

## rate_limited Decorator

The **rate_limited** decorator takes a rate limiter name as an argument and applies rate limiting to the decorated function.

import threading
import time
from functools import wraps

class RateLimiter:
    def __init__(self, rate, burst_size):
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        with self.lock:
            current_time = time.time()
            elapsed_time = current_time - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = current_time
            if self.tokens < 1:
                return False
            self.tokens -= 1
            return True

    def add_tokens(self, num_tokens):
        with self.lock:
            self.tokens = min(self.burst_size, self.tokens + num_tokens)

    def get_rate(self):
        return self.rate

    def get_burst_size(self):
        return self.burst_size

class RateLimiterManager:
    def __init__(self):
        self.rate_limiters = {}
        self.lock = threading.Lock()

    def get_rate_limiter(self, name):
        with self.lock:
            if name not in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' not found")
            return self.rate_limiters[name]

    def add_rate_limiter(self, name, rate, burst_size):
        with self.lock:
            if name in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' already exists")
            self.rate_limiters[name] = RateLimiter(rate, burst_size)

    def remove_rate_limiter(self, name):
        with self.lock:
            if name not in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' not found")
            del self.rate_limiters[name]

    def update_rate_limiter(self, name, rate, burst_size):
        with self.lock:
            if name not in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' not found")
            self.rate_limiters[name] = RateLimiter(rate, burst_size)

def rate_limited(rate_limiter_name):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            rate_limiter_manager = RateLimiterManager()
            rate_limiter = rate_limiter_manager.get_rate_limiter(rate_limiter_name)
            if not rate_limiter.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

# Example usage:
rate_limiter_manager = RateLimiterManager()
rate_limiter_manager.add_rate_limiter("my_rate_limiter", 5, 10)

@rate_limited("my_rate_limiter")
def my_function():

## Introduction

The rate limiter module is designed to provide a flexible and efficient way to limit the rate at which requests are processed. The module implements the token bucket algorithm, which allows for a configurable rate and burst size. The module also supports multiple named rate limiters, making it suitable for a variety of use cases, such as per-user or per-endpoint rate limiting.

## Module Components

The rate limiter module consists of the following components:

1. **RateLimiter**: The main class responsible for implementing the token bucket algorithm.
2. **RateLimiterManager**: A class responsible for managing multiple named rate limiters.
3. **rate_limited**: A decorator that can be used to easily apply rate limiting to functions.

## Token Bucket Algorithm

The token bucket algorithm works as follows:

1. Tokens are added to the bucket at a constant rate (configurable).
2. Each request consumes a token from the bucket.
3. If the bucket is empty, the request is blocked until a token is available.
4. The bucket has a maximum size (configurable), which allows for bursts of requests.

## RateLimiter Class

The **RateLimiter** class has the following methods:

* `__init__(rate, burst_size)`: Initializes the rate limiter with the specified rate and burst size.
* `allow_request()`: Checks if a request is allowed based on the current state of the token bucket.
* `add_tokens(num_tokens)`: Adds tokens to the bucket.
* `get_rate()`: Returns the current rate of the rate limiter.
* `get_burst_size()`: Returns the current burst size of the rate limiter.
* `reset()`: Resets the rate limiter to its initial state.

        Updates a rate limiter with the specified name, rate, and burst size.

        :param name: The name of the rate limiter
        :param rate: The rate at which tokens are added to the bucket (requests per second)
        :param burst_size: The maximum size of the token bucket
        :raises ValueError: If the rate limiter with the specified name does not exist
        """
        with self.lock:
            if name not in self.rate_limiters:
                raise ValueError(f"Rate limiter '{name}' not found")
            self.rate_limiters[name] = RateLimiter(rate, burst_size)

    def list_rate_limiters(self):
        """
        Returns a list of all rate limiters managed by the manager.

        :return: A list of all rate limiters managed by the manager
        """
        with self.lock:
            return list(self.rate_limiters.keys())

def rate_limited(rate_limiter_name, rate_limiter_manager=None):
    """
    A decorator that applies rate limiting to a function.

    :param rate_limiter_name: The name of the rate limiter to use
    :param rate_limiter_manager: The rate limiter manager to use (optional)
    :return: A decorator that applies rate limiting to a function
    """
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            if rate_limiter_manager is None:
                rate_limiter_manager = RateLimiterManager()
            rate_limiter = rate_limiter_manager.get_rate_limiter(rate_limiter_name)
            if not rate_limiter.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator
