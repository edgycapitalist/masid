import heapq
from task import Task

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.task_status = {}

    def add_task(self, task):
        heapq.heappush(self.queue, (task.priority, task))
        self.task_status[task.id] = task.status

    def get_task(self):
        if self.queue:
            _, task = heapq.heappop(self.queue)
            return task
        return None

    def update_task_status(self, task_id, status):
        self.task_status[task_id] = status

## Overview

This document outlines the architecture of an in-memory task queue implemented in Python. The task queue supports adding tasks with priorities, allows worker threads to consume tasks, provides task status tracking, and supports task retry with configurable max retries.

## Components

1. **Task**: Represents a single task with a unique ID, priority, and execution function.
2. **Task Queue**: Manages the addition, retrieval, and tracking of tasks.
3. **Worker Thread**: Consumes tasks from the task queue and executes them.
4. **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).

## Module Decomposition

The system will be decomposed into the following modules:

1. `task.py`: Defines the `Task` class.

5. `main.py`: Entry point for the application.

	* `get_task()`: Retrieves the next task from the queue.
	* `update_task_status(task_id, status)`: Updates the status of a task.
2. **Worker Thread Interface**:
	* `start()`: Starts the worker thread.
	* `stop()`: Stops the worker thread.
3. **Task Status Tracker Interface**:
	* `get_task_status(task_id)`: Retrieves the status of a task.

2. **Task Consumption**:

	* The `WorkerThread` executes the task and updates its status using the `update_task_status()` method.
3. **Task Status Tracking**:
