## Introduction
The in-memory task queue is designed to support adding tasks with priorities, allowing worker threads to consume tasks, providing task status tracking, and supporting task retry with configurable max retries. The system will also include a mechanism for graceful shutdown.

## High-Level Design
The task queue will be implemented using a combination of data structures and threading primitives in Python. The main components of the system are:

*   **Task**: Represents a single task with a unique ID, priority, and status.
*   **Task Queue**: A data structure that stores tasks and provides methods for adding, retrieving, and updating tasks.
*   **Worker**: A thread that consumes tasks from the task queue, executes them, and updates their status.
*   **Task Executor**: A component that manages the execution of tasks by worker threads.
*   **Shutdown Manager**: A component that handles the shutdown of the task queue and worker threads.

## Module Decomposition
The system will be decomposed into the following modules:

*   `task.py`: Defines the Task class and its methods.
*   `task_queue.py`: Implements the Task Queue data structure and its methods.
*   `worker.py`: Defines the Worker class and its methods.
*   `task_executor.py`: Manages the execution of tasks by worker threads.
*   `shutdown_manager.py`: Handles the shutdown of the task queue and worker threads.
*   `main.py`: The entry point of the application, responsible for creating and starting the task queue and worker threads.

## Interfaces
The following interfaces will be defined:

*   `TaskQueue`: Provides methods for adding, retrieving, and updating tasks.
*   `Worker`: Provides methods for consuming tasks and updating their status.
*   `TaskExecutor`: Provides methods for managing the execution of tasks by worker threads.
*   `ShutdownManager`: Provides methods for shutting down the task queue and worker threads.

## Data Flow
The data flow of the system is as follows:

1.  **Task Creation**: A task is created and added to the task queue.
2.  **Task Retrieval**: A worker thread retrieves a task from the task queue.
3.  **Task Execution**: The worker thread executes the task and updates its status.
4.  **Task Update**: The task queue is updated with the new status of the task.
5.  **Shutdown**: The shutdown manager is notified to shut down the task queue and worker threads.

The system will be implemented using Python 3.9 or later, with the following libraries:

*   `threading`: For creating and managing worker threads.
*   `queue`: For implementing the task queue data structure.
*   `enum`: For defining the task status enum.

from enum import Enum
from uuid import uuid4

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, priority, func, *args, **kwargs):
        self.id = uuid4()
        self.priority = priority
        self.status = TaskStatus.PENDING
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.max_retries = 3
        self.retries = 0

    def execute(self):
        try:
            self.func(*self.args, **self.kwargs)
            self.status = TaskStatus.COMPLETED
        except Exception as e:
            self.status = TaskStatus.FAILED
            self.retries += 1
            if self.retries < self.max_retries:
                # Retry the task
                self.status = TaskStatus.PENDING
            else:
                # Max retries exceeded, do not retry
                pass

import heapq
from queue import Queue

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.task_map = {}

    def add_task(self, task):
        heapq.heappush(self.queue, (task.priority, task.id))
        self.task_map[task.id] = task

    def get_task(self):
        if not self.queue:
            return None
        _, task_id = heapq.heappop(self.queue)
        return self.task_map[task_id]

    def update_task(self, task):
