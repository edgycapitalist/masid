**Architecture Document: In-Memory Task Queue**
=====================================================

**Overview**
------------

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and includes a mechanism for graceful shutdown.

**Components**
--------------

1. **Task**: Represents a single task with a unique ID, priority, and associated data.
2. **Task Queue**: A data structure that stores tasks in memory, allowing for efficient insertion and retrieval of tasks based on priority.
3. **Worker Thread**: Responsible for consuming tasks from the task queue, executing them, and updating their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

**Design**
----------

### Task Queue

The task queue is implemented as a priority queue using the `heapq` module in Python. This allows for efficient insertion and retrieval of tasks based on priority.

### Task

Each task has the following attributes:

* `id`: Unique task ID
* `priority`: Task priority (higher values indicate higher priority)
* `data`: Associated task data
* `status`: Task status (pending, running, completed, failed)
* `retries`: Number of retries attempted
* `max_retries`: Configurable max retries

### Worker Thread

Worker threads consume tasks from the task queue, execute them, and update their status. If a task fails, the worker thread will retry the task up to the configured max retries.

### Task Status Tracker

The task status tracker maintains the status of each task. It provides methods to update the status of a task and retrieve the status of a task by ID.

```python
import heapq
import threading
import time
from enum import Enum

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, data, max_retries):
        self.id = id
        self.priority = priority
        self.data = data
        self.status = TaskStatus.PENDING
        self.retries = 0
        self.max_retries = max_retries

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            return heapq.heappop(self.queue)[1]

class TaskStatusTracker:
    def __init__(self):
        self.status_map = {}

    def update_status(self, task, status):
        self.status_map[task.id] = status

    def get_status(self, task_id):
        return self.status_map.get(task_id)

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            self.execute_task(task)

    def execute_task(self, task):
        self.task_status_tracker.update_status(task, TaskStatus.RUNNING)
        try:
            # Simulate task execution
            time.sleep(1)
            self.task_status_tracker.update_status(task, TaskStatus.COMPLETED)
        except Exception as e:
            self.task_status_tracker.update_status(task, TaskStatus.FAILED)
            if task.retries < task.max_retries:
                task.retries += 1
                self.task_queue.add_task(task)

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()

    # Create worker threads
    worker_threads = []
    for _ in range(5):
        worker_thread = WorkerThread(task_queue, task_status_tracker)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Add tasks to the queue
    for i in range(10):
        task = Task(i, i, f"Task {i}", 3)
        task_queue.add_task(task)

    # Wait for all tasks to complete
    for worker_thread in worker_threads:
        worker_thread.join()

if __name__ == "__main__":
    main()
```

**Overview**
------------

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and includes a mechanism for graceful shutdown.

**Components**
--------------

1. **Task**: Represents a single task with a unique ID, priority, and associated data.
2. **Task Queue**: A data structure that stores tasks in memory, allowing for efficient insertion and retrieval of tasks based on priority.
3. **Worker Thread**: Responsible for consuming tasks from the task queue, executing them, and updating their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

**Design**
----------

### Task Queue

The task queue is implemented as a priority queue using the `heapq` module in Python. This allows for efficient insertion and retrieval of tasks based on priority.

### Task

Each task has the following attributes:

* `id`: Unique task ID
* `priority`: Task priority (higher values indicate higher priority)
* `data`: Associated task data
* `status`: Task status (pending, running, completed, failed)
* `retries`: Number of retries attempted
* `max_retries`: Configurable max retries
* `created_at`: Timestamp when the task was created
* `updated_at`: Timestamp when the task was last updated

### Worker Thread

Worker threads consume tasks from the task queue, execute them, and update their status. If a task fails, the worker thread will retry the task up to the configured max retries.

### Task Status Tracker

The task status tracker maintains the status of each task. It provides methods to update the status of a task and retrieve the status of a task by ID.

```python
        task.updated_at = datetime.now()

    def get_status(self, task_id):

        self.task_status_tracker = task_status_tracker

            if task is None:

            self.execute_task(task)

                self.task_queue.add_task(task)
```

**Overview**
------------

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and includes a mechanism for graceful shutdown.

**Components**
--------------

1. **Task**: Represents a single task with a unique ID, priority, and associated data.
2. **Task Queue**: A data structure that stores tasks in memory, allowing for efficient insertion and retrieval of tasks based on priority.
3. **Worker Thread**: Responsible for consuming tasks from the task queue, executing them, and updating their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

**Design**
----------

### Task Queue

The task queue is implemented as a priority queue using the `heapq` module in Python. This allows for efficient insertion and retrieval of tasks based on priority.

### Task

Each task has the following attributes:

* `id`: Unique task ID
* `priority`: Task priority (higher values indicate higher priority)
* `data`: Associated task data
* `status`: Task status (pending, running, completed, failed)
* `retries`: Number of retries attempted
* `max_retries`: Configurable max retries
* `created_at`: Timestamp when the task was created
* `updated_at`: Timestamp when the task was last updated

### Worker Thread

Worker threads consume tasks from the task queue, execute them, and update their status. If a task fails, the worker thread will retry the task up to the configured max retries.

### Task Status Tracker

The task status tracker maintains the status of each task. It provides methods to update the status of a task and retrieve the status of a task by ID.

```python
import heapq
import threading
import time
from enum import Enum
from datetime import datetime

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, id, priority, data, max_retries):
        self.id = id
        self.priority = priority
        self.data = data
        self.status = TaskStatus.PENDING
        self.retries = 0
        self.max_retries = max_retries
        self.created_at = datetime.now()
        self.updated_at = datetime.now()

    def __repr__(self):
        return f"Task(id={self.id}, priority={self.priority}, data={self.data})"

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            return heapq.heappop(self.queue)[1]

    def is_empty(self):
        with self.lock:
            return len(self.queue) == 0

class TaskStatusTracker:
    def __init__(self):
        self.status_map = {}

    def update_status(self, task, status):
        task.status = status
        task.updated_at = datetime.now()
        self.status_map[task.id] = status

    def get_status(self, task_id):
        return self.status_map.get(task_id)

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            self.execute_task(task)

    def execute_task(self, task):
        self.task_status_tracker.update_status(task, TaskStatus.RUNNING)
        try:
            # Simulate task execution
            time.sleep(1)
            self.task_status_tracker.update_status(task, TaskStatus.COMPLETED)
        except Exception as e:
            self.task_status_tracker.update_status(task, TaskStatus.FAILED)
            if task.retries < task.max_retries:
                task.retries += 1
                self.task_queue.add_task(task)

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()

    # Create worker threads
    worker_threads = []
    for _ in range(5):
        worker_thread = WorkerThread(task_queue, task_status_tracker)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Add tasks to the queue
    for i in range(10):
        task = Task(i, i, f"Task {i}", 3)
        task_queue.add_task(task)

    # Wait for all tasks to complete
    while not task_queue.is_empty():
        time.sleep(0.1)

    for worker_thread in worker_threads:
        worker_thread.join()

if __name__ == "__main__":
    main()
```

```python
    def __repr__(self):
        return f"Task(id={self.id}, priority={self.priority}, data={self.data})"

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            return heapq.heappop(self.queue)[1]

    def is_empty(self):
        with self.lock:
            return len(self.queue) == 0

class TaskStatusTracker:
    def __init__(self):
        self.status_map = {}

    def update_status(self, task, status):
        task.status = status
        task.updated_at = datetime.now()
        self.status_map[task.id] = status

    def get_status(self, task_id):
        return self.status_map.get(task_id)

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker
        self.stop_flag = threading.Event()

    def run(self):
        while not self.stop_flag.is_set():
            task = self.task_queue.get_task()
            if task is None:
                time.sleep(0.1)
                continue
            self.execute_task(task)

    def execute_task(self, task):
        try:
            self.task_status_tracker.update_status(task, TaskStatus.RUNNING)
            # Simulate task execution
            time.sleep(1)
            self.task_status_tracker.update_status(task, TaskStatus.COMPLETED)
        except Exception as e:
            self.task_status_tracker.update_status(task, TaskStatus.FAILED)
            if task.retries < task.max_retries:
                task.retries += 1
                self.task_queue.add_task(task)

    def stop(self):
        self.stop_flag.set()

def main():
    task_queue = TaskQueue()
    task_status_tracker = TaskStatusTracker()

    # Create worker threads
    worker_threads = []
    for _ in range(5):
        worker_thread = WorkerThread(task_queue, task_status_tracker)
        worker_thread.start()
        worker_threads.append(worker_thread)

    # Add tasks to the queue
    for i in range(10):
        task = Task(i, i, f"Task {i}", 3)
        task_queue.add_task(task)

    # Wait for all tasks to complete
    while not task_queue.is_empty():
        time.sleep(0.1)

    # Stop worker threads
    for worker_thread in worker_threads:
        worker_thread.stop()
        worker_thread.join()

if __name__ == "__main__":
    main()
```
