## Introduction

The goal of this project is to design and implement a Python data pipeline that reads CSV files with configurable schema validation, applies a chain of transformations, and outputs results to CSV or JSON. The pipeline should also support a plugin system for custom transformations and provide clear error reporting for malformed data.

## High-Level Architecture

The data pipeline will consist of the following components:

1. **Data Ingestion**: Responsible for reading CSV files with configurable schema validation.
2. **Transformation Engine**: Applies a chain of transformations (filtering, mapping, aggregation) to the ingested data.
3. **Plugin System**: Allows for custom transformations to be added to the pipeline.
4. **Data Output**: Outputs the transformed data to CSV or JSON.
5. **Error Handling**: Provides clear error reporting for malformed data.

## Component Design

### Data Ingestion

* The data ingestion component will use the `pandas` library to read CSV files.
* Schema validation will be performed using the `jsonschema` library.
* The component will take in a `schema` parameter that defines the expected structure of the CSV file.

### Transformation Engine

* The transformation engine will use a chain of functions to apply transformations to the data.
* Each transformation function will take in a `data` parameter and return the transformed data.
* The engine will support the following built-in transformations:
	+ Filtering: `filter_data`
	+ Mapping: `map_data`
	+ Aggregation: `aggregate_data`
* The engine will also support custom transformations through the plugin system.

### Plugin System

* The plugin system will allow developers to add custom transformations to the pipeline.
* Plugins will be implemented as Python functions that take in a `data` parameter and return the transformed data.
* Plugins will be registered with the transformation engine using a `register_plugin` function.

### Data Output

* The data output component will use the `pandas` library to write the transformed data to CSV or JSON.
* The component will take in a `format` parameter that specifies the output format (CSV or JSON).

### Error Handling

* The error handling component will use the `logging` library to log errors.
* The component will take in an `error` parameter that describes the error.
* The component will provide clear error reporting for malformed data.

## Implementation Details

The implementation will be written in Python 3.8+.
The `pandas` library will be used for data manipulation.
The `jsonschema` library will be used for schema validation.
The `logging` library will be used for error handling.

def filter_data(data: pd.DataFrame) -> pd.DataFrame:

def register_plugin(plugin: Callable[[pd.DataFrame], pd.DataFrame]) -> None:
    plugins.append(plugin)

    jsonschema.validate(instance=data.to_dict(orient="records"), schema=schema)
    return data

# Define the error handling component

    logging.error(error)

# Main function
def main() -> None:
    try:
        data = ingest_data("input.csv", schema)
        data = transform_data(data)
        output_data(data, "csv")
    except Exception as e:
        handle_error(str(e))

if __name__ == "__main__":

## Introduction

The goal of this project is to design and implement a Python data pipeline that reads CSV files with configurable schema validation, applies a chain of transformations (filtering, mapping, aggregation), and outputs results to CSV or JSON. The pipeline should also support a plugin system for custom transformations and provide clear error reporting for malformed data.

## High-Level Architecture

The data pipeline will consist of the following components:

1. **Data Ingestion**: Responsible for reading CSV files with configurable schema validation.
2. **Transformation Engine**: Applies a chain of transformations (filtering, mapping, aggregation) to the ingested data.
3. **Plugin System**: Allows for custom transformations to be added to the pipeline.
4. **Data Output**: Outputs the transformed data to CSV or JSON.
5. **Error Handling**: Provides clear error reporting for malformed data.

## Component Design

### Data Ingestion

* The data ingestion component will use the `pandas` library to read CSV files.
* Schema validation will be performed using the `jsonschema` library.
* The component will take in a `schema` parameter that defines the expected structure of the CSV file.
* The component will also handle missing values and duplicate values.

### Transformation Engine

* The transformation engine will use a chain of functions to apply transformations to the data.
* Each transformation function will take in a `data` parameter and return the transformed data.
* The engine will support the following built-in transformations:
	+ Filtering: `filter_data`
	+ Mapping: `map_data`
	+ Aggregation: `aggregate_data`
* The engine will also support custom transformations through the plugin system.

### Plugin System

* The plugin system will allow developers to add custom transformations to the pipeline.
* Plugins will be implemented as Python functions that take in a `data` parameter and return the transformed data.
* Plugins will be registered with the transformation engine using a `register_plugin` function.

### Data Output

* The data output component will use the `pandas` library to write the transformed data to CSV or JSON.
* The component will take in a `format` parameter that specifies the output format (CSV or JSON).

### Error Handling

* The error handling component will use the `logging` library to log errors.
* The component will take in an `error` parameter that describes the error.
* The component will provide clear error reporting for malformed data.

## Implementation Details

The implementation will be written in Python 3.8+.
The `pandas` library will be used for data manipulation.
The `jsonschema` library will be used for schema validation.
The `logging` library will be used for error handling.

```python
import pandas as pd
import jsonschema
import logging
from typing import Callable, List

# Define the schema for the CSV file
schema = {
    "type": "object",
    "properties": {
        "name": {"type": "string"},
        "age": {"type": "integer"}
    },
    "required": ["name", "age"]
}

# Define the transformation functions
def filter_data(data: pd.DataFrame) -> pd.DataFrame:
    return data[data["age"] > 18]

def map_data(data: pd.DataFrame) -> pd.DataFrame:
    data["name"] = data["name"].str.upper()
    return data

def aggregate_data(data: pd.DataFrame) -> pd.DataFrame:
    return data.groupby("name").count()

# Define the plugin system
def register_plugin(plugin: Callable[[pd.DataFrame], pd.DataFrame]) -> None:
    plugins.append(plugin)

plugins = []

# Define the data ingestion component
def ingest_data(file_path: str, schema: dict) -> pd.DataFrame:
    try:
        data = pd.read_csv(file_path)
        jsonschema.validate(instance=data.to_dict(orient="records"), schema=schema)
        return data
    except jsonschema.ValidationError as e:
        logging.error(f"Schema validation error: {e}")
        raise
    except pd.errors.EmptyDataError:
        logging.error("No data in file")
        raise
    except pd.errors.ParserError as e:
        logging.error(f"Error parsing file: {e}")
        raise

# Define the transformation engine
def transform_data(data: pd.DataFrame) -> pd.DataFrame:
    data = filter_data(data)
    data = map_data(data)
    data = aggregate_data(data)
    for plugin in plugins:
        data = plugin(data)
    return data

# Define the data output component
def output_data(data: pd.DataFrame, format: str) -> None:
    if format == "csv":
        data.to_csv("output.csv", index=False)
    elif format == "json":
        data.to_json("output.json", orient="records")

# Define the error handling component
def handle_error(error: str) -> None:
    logging.error(error)

# Main function
def main() -> None:
    try:
        data = ingest_data("input.csv", schema)
        data = transform_data(data)
        output_data(data, "csv")
    except Exception as e:
        handle_error(str(e))

if __name__ == "__main__":
    main()
