**Architecture Document: In-Memory Task Queue**
=====================================================

### Introduction

The in-memory task queue is a critical component of our project, responsible for managing tasks with varying priorities, tracking their status, and ensuring reliable execution. This document outlines the high-level design of the task queue, including its components, interfaces, and interactions.

### Components

1. **Task**: Represents a single unit of work with a unique identifier, priority, and execution function.
2. **Task Queue**: Manages the task lifecycle, including adding, consuming, and tracking task status.
3. **Worker Thread**: Executes tasks from the task queue.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

### Interfaces

1. **Task Queue Interface**:
	* `add_task(task: Task)`: Adds a new task to the queue.
	* `get_task()`: Retrieves the next task from the queue based on priority.
	* `update_task_status(task_id: str, status: str)`: Updates the status of a task.
2. **Worker Thread Interface**:
	* `execute_task(task: Task)`: Executes a task.
	* `report_task_status(task_id: str, status: str)`: Reports the status of a task after execution.

### Data Flow

1. **Task Addition**:
	* Client adds a new task to the task queue using the `add_task` method.
	* Task queue stores the task with its priority and initial status (pending).
2. **Task Consumption**:
	* Worker thread retrieves the next task from the task queue using the `get_task` method.
	* Worker thread executes the task using the `execute_task` method.
	* Worker thread reports the task status using the `report_task_status` method.
3. **Task Status Tracking**:
	* Task status tracker updates the task status based on the report from the worker thread.

### Technology Choices

* **Python**: As the primary programming language.
* **Threading**: For worker threads to execute tasks concurrently.
* **Queue**: For storing tasks in memory.

import threading
from queue import PriorityQueue
from typing import Callable, Dict

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable, max_retries: int = 3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = "pending"

class TaskQueue:
    def __init__(self):
        self.queue = PriorityQueue()
        self.task_status: Dict[str, str] = {}
        self.lock = threading.Lock()

    def add_task(self, task: Task):
        with self.lock:
            self.queue.put((task.priority, task))
            self.task_status[task.task_id] = task.status

    def get_task(self) -> Task:
        with self.lock:
            if self.queue.empty():
                return None
            _, task = self.queue.get()
            return task

    def update_task_status(self, task_id: str, status: str):
        with self.lock:
            self.task_status[task_id] = status

class WorkerThread(threading.Thread):
    def __init__(self, task_queue: TaskQueue):
        super().__init__()
        self.task_queue = task_queue

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task is None:
                break
            try:
                task.func()
                self.task_queue.update_task_status(task.task_id, "completed")
            except Exception as e:
                task.retries += 1
                if task.retries < task.max_retries:
                    self.task_queue.add_task(task)
                    self.task_queue.update_task_status(task.task_id, "failed")
                else:
                    self.task_queue.update_task_status(task.task_id, "failed")

### Introduction

The in-memory task queue is a critical component of our project, responsible for managing tasks with varying priorities, tracking their status, and ensuring reliable execution. This document outlines the high-level design of the task queue, including its components, interfaces, and interactions.

### Components

1.  **Task**: Represents a single unit of work with a unique identifier, priority, and execution function.
2.  **Task Queue**: Manages the task lifecycle, including adding, consuming, and tracking task status.
3.  **Worker Thread**: Executes tasks from the task queue.
4.  **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

### Interfaces

1.  **Task Queue Interface**:
    *   `add_task(task: Task)`: Adds a new task to the queue.
    *   `get_task()`: Retrieves the next task from the queue based on priority.
    *   `update_task_status(task_id: str, status: str)`: Updates the status of a task.
    *   `get_task_status(task_id: str)`: Retrieves the status of a task.
2.  **Worker Thread Interface**:
    *   `execute_task(task: Task)`: Executes a task.
    *   `report_task_status(task_id: str, status: str)`: Reports the status of a task after execution.

### Data Flow

1.  **Task Addition**:
    *   Client adds a new task to the task queue using the `add_task` method.
    *   Task queue stores the task with its priority and initial status (pending).
2.  **Task Consumption**:
    *   Worker thread retrieves the next task from the task queue using the `get_task` method.
    *   Worker thread executes the task using the `execute_task` method.
    *   Worker thread reports the task status using the `report_task_status` method.
3.  **Task Status Tracking**:
    *   Task status tracker updates the task status based on the report from the worker thread.

### Technology Choices

*   **Python**: As the primary programming language.

import threading
from queue import PriorityQueue
from typing import Callable, Dict
import time

class Task:
    def __init__(self, task_id: str, priority: int, func: Callable, max_retries: int = 3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = "pending"

class TaskQueue:
    def __init__(self, max_size: int = 100, num_workers: int = 5):
        self.queue = PriorityQueue()
        self.task_status: Dict[str, str] = {}
        self.lock = threading.Lock()
        self.max_size = max_size
        self.num_workers = num_workers
        self.workers = []

    def add_task(self, task: Task):
        with self.lock:
            if self.queue.qsize() < self.max_size:
                self.queue.put((task.priority, task))
                self.task_status[task.task_id] = task.status
            else:
                print("Task queue is full. Task not added.")

    def get_task(self) -> Task:
        with self.lock:
            if self.queue.empty():
                return None
            _, task = self.queue.get()
            return task

    def update_task_status(self, task_id: str, status: str):
        with self.lock:
            self.task_status[task_id] = status

    def get_task_status(self, task_id: str) -> str:
        with self.lock:
            return self.task_status.get(task_id)

    def start_workers(self):
        for _ in range(self.num_workers):
            worker = WorkerThread(self)
            worker.start()
            self.workers.append(worker)

    def stop_workers(self):
        for worker in self.workers:
            worker.stop()
            worker.join()
