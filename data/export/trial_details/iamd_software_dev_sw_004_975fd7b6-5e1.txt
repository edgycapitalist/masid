**Architecture Document: In-Memory Task Queue**

## Introduction

The in-memory task queue is designed to manage tasks with priorities, allow worker threads to consume tasks, track task status, and support task retry with configurable max retries. The system will also provide a mechanism for graceful shutdown.

## Components

1. **Task**: Represents a single task with a unique ID, priority, and a callable function to be executed.
2. **Task Queue**: A data structure that stores tasks in a priority order, allowing worker threads to consume tasks.
3. **Worker Thread**: Responsible for consuming tasks from the task queue and executing them.
4. **Task Status Tracker**: Tracks the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Allows tasks to be retried with a configurable max retries.

## System Design

### Task Queue

The task queue will be implemented using a priority queue data structure, which will store tasks based on their priority. The priority queue will be implemented using a binary heap, which will ensure efficient insertion and removal of tasks.

### Worker Thread

Worker threads will consume tasks from the task queue and execute them. Each worker thread will have its own task queue, which will be populated with tasks from the main task queue. This will ensure that tasks are executed in parallel and that the main task queue is not blocked.

### Task Status Tracker

The task status tracker will be implemented using a dictionary, which will store the status of each task. The status will be updated by the worker threads as they execute tasks.

### Retry Mechanism

The retry mechanism will be implemented using a decorator, which will wrap the task function and retry it if it fails. The max retries will be configurable, and the task will be marked as failed if it exceeds the max retries.

import threading
import heapq
import time
from functools import wraps

class Task:
    def __init__(self, id, priority, func, *args, **kwargs):
        self.id = id
        self.priority = priority
        self.func = func
        self.args = args
        self.kwargs = kwargs
        self.status = 'pending'
        self.retries = 0

    def execute(self):
        try:
            self.func(*self.args, **self.kwargs)
            self.status = 'completed'
        except Exception as e:
            self.status = 'failed'
            self.retries += 1
            raise e

class TaskQueue:
    def __init__(self):
        self.queue = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.queue, (task.priority, task))
            print(f'Task {task.id} added to queue')

    def get_task(self):
        with self.lock:
            if not self.queue:
                return None
            _, task = heapq.heappop(self.queue)
            return task

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                task.status = 'running'
                try:
                    task.execute()
                except Exception as e:
                    print(f'Task {task.id} failed with error: {e}')
                    if task.retries < 3:  # max retries
                        self.task_queue.add_task(task)
                    else:
                        print(f'Task {task.id} failed after max retries')
            else:
                time.sleep(1)

    def stop(self):
        self.running = False

def retry(max_retries):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            retries = 0
            while retries < max_retries:
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f'Error: {e}')
                    retries += 1
            raise Exception(f'Max retries exceeded')
        return wrapper
    return decorator

# Example usage:

worker.start()

@retry(3)
def example_task():
