**Architecture Document: Rate Limiter Module**

## Introduction
The rate limiter module is designed to control the frequency of requests to a system, preventing abuse and ensuring fair usage. This module implements the token bucket algorithm, allowing for configurable rates and burst sizes. It supports multiple named rate limiters and provides a decorator for easy application to functions. The module is designed to be thread-safe, ensuring reliable operation in concurrent environments.

## Module Structure
The rate limiter module will consist of the following components:

1. **RateLimiter**: The core class responsible for managing the token bucket and enforcing rate limits.
2. **RateLimiterManager**: A class that manages multiple named rate limiters, providing a centralized registry and access to rate limiters.
3. **rate_limited**: A decorator that applies rate limiting to functions.

## Token Bucket Algorithm
The token bucket algorithm is a widely used approach for rate limiting. It works as follows:

* A bucket is filled with tokens at a constant rate (requests per second).
* When a request is made, a token is removed from the bucket.
* If the bucket is empty, the request is blocked until a token is available.
* The burst size determines the maximum number of tokens that can be stored in the bucket, allowing for short-term spikes in traffic.

## RateLimiter Class
The RateLimiter class will have the following attributes and methods:

* **`__init__(rate, burst_size)`**: Initializes the rate limiter with the specified rate (requests per second) and burst size.
* **`allow_request()`**: Attempts to remove a token from the bucket, returning `True` if successful and `False` otherwise.
* **`update_tokens()`**: Updates the token count based on the elapsed time since the last update.

## RateLimiterManager Class
The RateLimiterManager class will have the following attributes and methods:

* **`__init__()`**: Initializes the rate limiter manager with an empty registry.
* **`get_rate_limiter(name)`**: Retrieves a rate limiter by name, creating a new one if it doesn't exist.
* **`add_rate_limiter(name, rate_limiter)`**: Adds a rate limiter to the registry.

## rate_limited Decorator
The rate_limited decorator will have the following attributes and methods:

* **`__init__(name, rate_limiter_manager)`**: Initializes the decorator with the name of the rate limiter and the rate limiter manager.
* **`__call__(func)`**: Applies rate limiting to the decorated function.

import threading
import time
from functools import wraps

class RateLimiter:
    def __init__(self, rate, burst_size):
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        with self.lock:
            elapsed_time = time.time() - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = time.time()
            if self.tokens >= 1:
                self.tokens -= 1
                return True
            return False

    def update_tokens(self):
        with self.lock:
            elapsed_time = time.time() - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = time.time()

class RateLimiterManager:
    def __init__(self):
        self.registry = {}
        self.lock = threading.Lock()

    def get_rate_limiter(self, name):
        with self.lock:
            if name not in self.registry:
                self.registry[name] = RateLimiter(1, 10)  # default rate and burst size
            return self.registry[name]

    def add_rate_limiter(self, name, rate_limiter):
        with self.lock:
            self.registry[name] = rate_limiter

def rate_limited(name, rate_limiter_manager):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            rate_limiter = rate_limiter_manager.get_rate_limiter(name)
            if not rate_limiter.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

# Example usage
rate_limiter_manager = RateLimiterManager()
rate_limiter_manager.add_rate_limiter("example", RateLimiter(5, 10))  # 5 requests per second, burst size 10

@rate_limited("example", rate_limiter_manager)
def example_function():
    print("Hello, world!")

example_function()

## Introduction
The rate limiter module is designed to control the frequency of requests to a system, preventing abuse and ensuring fair usage. This module implements the token bucket algorithm, allowing for configurable rates and burst sizes. It supports multiple named rate limiters and provides a decorator for easy application to functions. The module is designed to be thread-safe, ensuring reliable operation in concurrent environments.

## Module Structure
The rate limiter module will consist of the following components:

1. **RateLimiter**: The core class responsible for managing the token bucket and enforcing rate limits.
2. **RateLimiterManager**: A class that manages multiple named rate limiters, providing a centralized registry and access to rate limiters.
3. **rate_limited**: A decorator that applies rate limiting to functions.

## Token Bucket Algorithm
The token bucket algorithm is a widely used approach for rate limiting. It works as follows:

* A bucket is filled with tokens at a constant rate (requests per second).
* When a request is made, a token is removed from the bucket.
* If the bucket is empty, the request is blocked until a token is available.
* The burst size determines the maximum number of tokens that can be stored in the bucket, allowing for short-term spikes in traffic.

## RateLimiter Class
The RateLimiter class will have the following attributes and methods:

* **`__init__(rate, burst_size)`**: Initializes the rate limiter with the specified rate (requests per second) and burst size.
* **`allow_request()`**: Attempts to remove a token from the bucket, returning `True` if successful and `False` otherwise.
* **`update_tokens()`**: Updates the token count based on the elapsed time since the last update.
* **`reset()`**: Resets the rate limiter to its initial state.

## RateLimiterManager Class
The RateLimiterManager class will have the following attributes and methods:

* **`__init__()`**: Initializes the rate limiter manager with an empty registry.
* **`get_rate_limiter(name)`**: Retrieves a rate limiter by name, creating a new one if it doesn't exist.
* **`add_rate_limiter(name, rate_limiter)`**: Adds a rate limiter to the registry.
* **`remove_rate_limiter(name)`**: Removes a rate limiter from the registry.
* **`list_rate_limiters()`**: Returns a list of all rate limiters in the registry.

## rate_limited Decorator
The rate_limited decorator will have the following attributes and methods:

* **`__init__(name, rate_limiter_manager, error_message=None)`**: Initializes the decorator with the name of the rate limiter and the rate limiter manager.
* **`__call__(func)`**: Applies rate limiting to the decorated function.
