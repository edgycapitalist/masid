**Architecture Document: In-Memory Task Queue**

## Overview
The in-memory task queue is designed to manage tasks with varying priorities, track their status, and provide a mechanism for worker threads to consume tasks. The system will support task retry with a configurable maximum number of retries and ensure a graceful shutdown.

## Components
1. **Task**: Represents a single task with a unique identifier, priority, and a callable function to execute.
2. **Task Queue**: An in-memory data structure that stores tasks and provides methods for adding, retrieving, and updating tasks.
3. **Worker Thread**: Responsible for consuming tasks from the task queue, executing them, and updating their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed) and provides methods for updating and querying task status.

## System Flow
1. **Task Addition**:
	* A task is created with a unique identifier, priority, and a callable function.
	* The task is added to the task queue.
2. **Task Consumption**:
	* A worker thread retrieves a task from the task queue based on its priority.
	* The worker thread executes the task and updates its status to "running".
	* If the task execution fails, the worker thread updates the task status to "failed" and schedules a retry if the maximum number of retries is not exceeded.
	* If the task execution succeeds, the worker thread updates the task status to "completed".
3. **Task Retry**:
	* If a task fails and the maximum number of retries is not exceeded, the task is rescheduled with a new execution time.
	* The task is updated in the task queue with a new priority based on its retry count.
4. **Shutdown**:
	* The system receives a shutdown signal.
	* All worker threads are stopped, and any in-progress tasks are cancelled.
	* The task queue is cleared, and all resources are released.

import threading
import queue
from enum import Enum
import time

class TaskStatus(Enum):
    PENDING = 1
    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

class Task:
    def __init__(self, task_id, priority, func, max_retries=3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()

    def add_task(self, task):
        self.queue.put((task.priority, task))

    def get_task(self):
        if not self.queue.empty():
            return self.queue.get()[1]
        return None

class WorkerThread(threading.Thread):
    def __init__(self, task_queue):
        super().__init__()
        self.task_queue = task_queue
        self.running = True

    def run(self):
        while self.running:
            task = self.task_queue.get_task()
            if task:
                task.status = TaskStatus.RUNNING
                try:
                    task.func()
                    task.status = TaskStatus.COMPLETED
                except Exception as e:
                    task.status = TaskStatus.FAILED
                    if task.retries < task.max_retries:
                        task.retries += 1
                        self.task_queue.add_task(task)
                    print(f"Task {task.task_id} failed with error: {str(e)}")
            time.sleep(1)

    def stop(self):
        self.running = False

class TaskStatusTracker:
    def __init__(self):
        self.tasks = {}

    def update_task_status(self, task):
        self.tasks[task.task_id] = task.status

    def get_task_status(self, task_id):
        return self.tasks.get(task_id)

import queue

class TaskStatus(Enum):

    RUNNING = 2
    COMPLETED = 3
    FAILED = 4

    def __init__(self, task_id, priority, func, max_retries=3):
        self.task_id = task_id
        self.priority = priority
        self.func = func
        self.max_retries = max_retries
        self.retries = 0
        self.status = TaskStatus.PENDING

class TaskQueue:
    def __init__(self):
        self.queue = queue.PriorityQueue()
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.queue.put((task.priority, task))

    def get_task(self):
        with self.lock:
            if not self.queue.empty():
                return self.queue.get()[1]
            return None

    def is_empty(self):
        with self.lock:
            return self.queue.empty()

                task.status = TaskStatus.RUNNING

                    task.status = TaskStatus.FAILED

                        task.retries += 1
                        self.task_queue.add_task(task)
                time.sleep(0.1)

### Architecture Document
The task queue system consists of the following components:

*   **Task**: Represents a task that needs to be executed. Each task has a unique `task_id`, a `priority`, a `func` attribute that represents the function to be executed, and a `status` attribute that represents the current status of the task.
*   **TaskQueue**: Represents a queue of tasks that need to be executed. The task queue is implemented as a priority queue, where tasks with higher priorities are executed first.
*   **WorkerThread**: Represents a thread that executes tasks from the task queue. Each worker thread runs in an infinite loop, executing tasks from the task queue until it is stopped.
*   **TaskStatusTracker**: Represents a tracker that keeps track of the status of each task. The task status tracker is used to update the status of each task as it is executed.

The system works as follows:

1.  A task is created and added to the task queue.
2.  A worker thread is started, which executes tasks from the task queue.
3.  As each task is executed, its status is updated in the task status tracker.
4.  The worker thread continues to execute tasks from the task queue until it is stopped.
5.  The task status tracker can be used to retrieve the status of each task at any time.

The system provides the following benefits:

*   **Decoupling**: The task queue system decouples the task creation and execution, allowing tasks to be created and executed independently.
*   **Prioritization**: The task queue system allows tasks to be prioritized, ensuring that high-priority tasks are executed first.
*   **Status tracking**: The task status tracker provides a way to track the status of each task, allowing for real-time monitoring and debugging.

Overall, the task queue system provides a flexible and scalable way to execute tasks in a concurrent and parallel manner.
