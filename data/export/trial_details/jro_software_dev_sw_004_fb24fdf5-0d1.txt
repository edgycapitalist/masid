**Architecture Document: In-Memory Task Queue**

### Overview

The in-memory task queue is designed to manage tasks with priorities, support worker threads, and provide task status tracking. The system will be implemented in Python, utilizing a thread-safe data structure to ensure efficient and safe task management.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and a callable function.
2. **Task Queue**: An in-memory data structure that stores tasks with their respective priorities and statuses.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their statuses.
4. **Task Status**: Enumerates the possible statuses of a task (pending, running, completed, failed).
5. **Task Retry Mechanism**: Allows tasks to be retried with a configurable maximum number of retries.

### System Design

The system consists of the following components:

*   **Task Queue**: Implemented as a priority queue using a `heapq` data structure to ensure efficient task retrieval based on priority.
*   **Task Manager**: Responsible for managing task statuses, updating task queues, and handling worker thread interactions.
*   **Worker Thread**: Implemented as a separate thread that consumes tasks from the task queue, executes them, and updates their statuses.
*   **Task Status Tracker**: Maintains the status of each task and updates it accordingly.

### Data Flow

1.  **Task Addition**:
    *   A new task is created with a unique ID, priority, and a callable function.
    *   The task is added to the task queue with its initial status set to "pending".
2.  **Worker Thread Execution**:
    *   A worker thread consumes a task from the task queue based on its priority.
    *   The task's status is updated to "running".
    *   The worker thread executes the task's callable function.
    *   If the task execution is successful, the task's status is updated to "completed".
    *   If the task execution fails, the task's status is updated to "failed" and the task is added back to the task queue for potential retry.
3.  **Task Retry Mechanism**:
    *   If a task fails, it is added back to the task queue with its retry count incremented.
    *   If the task's retry count exceeds the maximum allowed retries, its status is updated to "failed" and it is removed from the task queue.

### Technology Choices

*   **Python**: The primary programming language used for implementation.
*   **heapq**: A Python library used for implementing the priority queue data structure.
*   **threading**: A Python library used for creating and managing worker threads.
*   **logging**: A Python library used for logging system events and errors.

import heapq
import threading
import logging
import time

# Define task status enumerations
class TaskStatus:
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"

# Define the Task class
class Task:
    def __init__(self, id, priority, func):
        self.id = id
        self.priority = priority
        self.func = func
        self.status = TaskStatus.PENDING
        self.retry_count = 0

# Define the Task Queue class
class TaskQueue:
    def __init__(self):
        self.tasks = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            heapq.heappush(self.tasks, (task.priority, task))

    def get_task(self):
        with self.lock:
            if self.tasks:
                return heapq.heappop(self.tasks)[1]
            return None

# Define the Task Manager class
class TaskManager:
    def __init__(self):
        self.task_queue = TaskQueue()
        self.worker_threads = []
        self.max_retries = 3

    def add_task(self, task):
        self.task_queue.add_task(task)

    def start_worker_thread(self):
        thread = threading.Thread(target=self.worker_thread_func)
        thread.start()
        self.worker_threads.append(thread)

    def worker_thread_func(self):
        while True:
            task = self.task_queue.get_task()
            if task:
                task.status = TaskStatus.RUNNING
                try:
                    task.func()
                    task.status = TaskStatus.COMPLETED
                except Exception as e:
                    task.status = TaskStatus.FAILED
                    task.retry_count += 1
                    if task.retry_count < self.max_retries:
                        self.task_queue.add_task(task)
                    else:
                        logging.error(f"Task {task.id} failed after {self.max_retries} retries")
            else:
                time.sleep(1)

# Define the main function
def main():
    task_manager = TaskManager()

    # Create example tasks
    task1 = Task(1, 1, lambda: print("Task 1 executed"))
    task2 = Task(2, 2, lambda: print("Task 2 executed"))
    task3 = Task(3, 3, lambda: print("Task 3 executed"))

    # Add tasks to the task queue
    task_manager.add_task(task1)
    task_manager.add_task(task2)
    task_manager.add_task(task3)

    # Start worker threads
    task_manager.start_worker_thread()
    task_manager.start_worker_thread()

    # Wait for 10 seconds
    time.sleep(10)

if __name__ == "__main__":
    main()

### Overview

The in-memory task queue is designed to manage tasks with priorities, support worker threads, and provide task status tracking. The system will be implemented in Python, utilizing a thread-safe data structure to ensure efficient and safe task management.

### Components

1.  **Task**: Represents a single task with a unique ID, priority, and a callable function.
2.  **Task Queue**: An in-memory data structure that stores tasks with their respective priorities and statuses. The task queue includes methods to add, remove, and get tasks.
3.  **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their statuses. The worker threads can be stopped based on a specific condition.
4.  **Task Status**: Enumerates the possible statuses of a task (pending, running, completed, failed).
5.  **Task Retry Mechanism**: Allows tasks to be retried with a configurable maximum number of retries based on the task's status.

### System Design

The system consists of the following components:

*   **Task Queue**: Implemented as a priority queue using a `heapq` data structure to ensure efficient task retrieval based on priority.
*   **Task Manager**: Responsible for managing task statuses, updating task queues, and handling worker thread interactions.
*   **Worker Thread**: Implemented as a separate thread that consumes tasks from the task queue, executes them, and updates their statuses.
*   **Task Status Tracker**: Maintains the status of each task and updates it accordingly.

### Data Flow

1.  **Task Addition**:
    *   A new task is created with a unique ID, priority, and a callable function.
    *   The task is added to the task queue with its initial status set to "pending".
2.  **Worker Thread Execution**:
    *   A worker thread consumes a task from the task queue based on its priority.
    *   The task's status is updated to "running".
    *   The worker thread executes the task's callable function.
    *   If the task execution is successful, the task's status is updated to "completed".
    *   If the task execution fails, the task's status is updated to "failed" and the task is added back to the task queue for potential retry.
3.  **Task Retry Mechanism**:
    *   If a task fails, it is added back to the task queue with its retry count incremented.
    *   If the task's retry count exceeds the maximum allowed retries, its status is updated to "failed" and it is removed from the task queue.

import heapq
import threading
import logging
import time
import unittest

# Define task status enumerations
class TaskStatus:
    PENDING = "pending"
    RUNNING = "running"
    COMPLETED = "completed"
    FAILED = "failed"

class Task:
    def __init__(self, id, priority, func):
        self.id = id
        self.priority = priority
        self.func = func
        self.status = TaskStatus.PENDING
        self.retry_count = 0

# Define the Task Queue class
class TaskQueue:
    def __init__(self):
        self.tasks = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            try:
                heapq.heappush(self.tasks, (task.priority, task))
            except Exception as e:
                logging.error(f"Error adding task to queue: {e}")

    def get_task(self):
        with self.lock:
            try:
                if self.tasks:
                    return heapq.heappop(self.tasks)[1]
                return None
            except Exception as e:
                logging.error(f"Error getting task from queue: {e}")

    def remove_task(self, task_id):
        with self.lock:
            try:
                self.tasks = [(priority, task) for priority, task in self.tasks if task.id != task_id]
                heapq.heapify(self.tasks)
            except Exception as e:
                logging.error(f"Error removing task from queue: {e}")

# Define the Task Manager class
class TaskManager:
    def __init__(self):
        self.task_queue = TaskQueue()
        self.worker_threads = []
        self.max_retries = 3
        self.stopped = False

    def add_task(self, task):
        self.task_queue.add_task(task)

    def start_worker_thread(self):
        thread = threading.Thread(target=self.worker_thread_func)
        thread.start()
        self.worker_threads.append(thread)

    def stop_worker_threads(self):
        self.stopped = True
        for thread in self.worker_threads:
            thread.join()

    def worker_thread_func(self):
        while not self.stopped:

            if task:

                    else:

                        self.task_queue.remove_task(task.id)
            else:
                time.sleep(1)
