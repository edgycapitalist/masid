**Architecture Document: Rate Limiter Module**

### Overview

The rate limiter module is designed to limit the number of requests that can be made to a system within a given time frame. This module uses the token bucket algorithm to implement rate limiting. The module supports multiple named rate limiters, allowing for different rate limits to be applied to different users or endpoints.

### Components

*   **RateLimiter**: This class represents a single rate limiter. It has the following attributes:
    *   `name`: The name of the rate limiter.
    *   `rate`: The rate at which tokens are added to the bucket (requests per second).
    *   `burst_size`: The maximum number of tokens in the bucket.
    *   `tokens`: The current number of tokens in the bucket.
    *   `last_update`: The last time the bucket was updated.
*   **RateLimiterManager**: This class manages multiple rate limiters. It has the following attributes:
    *   `rate_limiters`: A dictionary of rate limiters, keyed by name.

### Interfaces

*   **RateLimiter**:
    *   `allow_request()`: Returns `True` if a request is allowed, `False` otherwise.
    *   `reset()`: Resets the rate limiter to its initial state.
*   **RateLimiterManager**:
    *   `get_rate_limiter(name)`: Returns the rate limiter with the given name.
    *   `add_rate_limiter(name, rate, burst_size)`: Adds a new rate limiter with the given name, rate, and burst size.

### Data Flow

1.  The `RateLimiterManager` class is instantiated and used to manage multiple rate limiters.
2.  The `RateLimiter` class is used to create individual rate limiters with specific rates and burst sizes.
3.  When a request is made, the `allow_request()` method of the corresponding rate limiter is called to determine if the request is allowed.
4.  If the request is allowed, the rate limiter updates its internal state to reflect the new number of tokens in the bucket.
5.  If the request is not allowed, the rate limiter returns `False`, indicating that the request should be blocked.

### Technology Choices

*   Python 3.8 or later is used as the programming language.
*   The `threading` module is used to ensure thread-safe operation.
*   The `time` module is used to keep track of time and update the rate limiter's internal state.

import threading
import time
from collections import defaultdict

class RateLimiter:
    def __init__(self, name, rate, burst_size):
        self.name = name
        self.rate = rate
        self.burst_size = burst_size
        self.tokens = burst_size
        self.last_update = time.time()
        self.lock = threading.Lock()

    def allow_request(self):
        with self.lock:
            current_time = time.time()
            elapsed_time = current_time - self.last_update
            self.tokens = min(self.burst_size, self.tokens + elapsed_time * self.rate)
            self.last_update = current_time
            if self.tokens < 1:
                return False
            self.tokens -= 1
            return True

    def reset(self):
        with self.lock:
            self.tokens = self.burst_size
            self.last_update = time.time()

class RateLimiterManager:
    def __init__(self):
        self.rate_limiters = {}
        self.lock = threading.Lock()

    def get_rate_limiter(self, name):
        with self.lock:
            return self.rate_limiters.get(name)

    def add_rate_limiter(self, name, rate, burst_size):
        with self.lock:
            self.rate_limiters[name] = RateLimiter(name, rate, burst_size)

def rate_limited(rate_limiter_name, rate_limiter_manager):
    def decorator(func):
        def wrapper(*args, **kwargs):
            rate_limiter = rate_limiter_manager.get_rate_limiter(rate_limiter_name)
            if rate_limiter is None:
                raise ValueError(f"Rate limiter '{rate_limiter_name}' not found")
            if not rate_limiter.allow_request():
                raise Exception("Rate limit exceeded")
            return func(*args, **kwargs)
        return wrapper
    return decorator

# Example usage:
