**Architecture Document: In-Memory Task Queue**

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and provides a graceful shutdown mechanism.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and payload.
2. **Task Queue**: An in-memory data structure that stores tasks and manages their priorities.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Configurable max retries for failed tasks.

### System Flow

1. **Task Addition**:
	* Client adds a task with a priority to the task queue.
	* Task is stored in the task queue with a unique ID.
2. **Task Consumption**:
	* Worker thread consumes the highest-priority task from the task queue.
	* Task is marked as "running" in the task status tracker.
3. **Task Execution**:
	* Worker thread executes the task.
	* If the task fails, it is marked as "failed" and the retry mechanism is triggered.
	* If the task succeeds, it is marked as "completed".
4. **Task Retry**:
	* If a task fails, it is retried up to the configurable max retries.
	* If all retries fail, the task is marked as "failed".
5. **Graceful Shutdown**:
	* The system is notified of an impending shutdown.
	* All worker threads are stopped, and any remaining tasks are marked as "failed".

import threading
import queue
import time

class Task:
    def __init__(self, id, priority, payload):
        self.id = id
        self.priority = priority
        self.payload = payload
        self.status = "pending"
        self.retries = 0

class TaskQueue:
    def __init__(self):
        self.tasks = []
        self.lock = threading.Lock()

    def add_task(self, task):
        with self.lock:
            self.tasks.append(task)
            self.tasks.sort(key=lambda x: x.priority, reverse=True)

    def get_task(self):
        with self.lock:
            if self.tasks:
                return self.tasks.pop(0)
            return None

class WorkerThread(threading.Thread):
    def __init__(self, task_queue, task_status_tracker):
        super().__init__()
        self.task_queue = task_queue
        self.task_status_tracker = task_status_tracker

    def run(self):
        while True:
            task = self.task_queue.get_task()
            if task:
                task.status = "running"
                self.task_status_tracker.update_task_status(task)
                try:
                    # Execute task
                    print(f"Executing task {task.id}")
                    time.sleep(1)  # Simulate task execution
                    task.status = "completed"
                except Exception as e:
                    task.status = "failed"
                    task.retries += 1
                    if task.retries < 3:  # Configurable max retries
                        self.task_queue.add_task(task)
                    else:
                        task.status = "failed"
                finally:
                    self.task_status_tracker.update_task_status(task)

class TaskStatusTracker:
    def __init__(self):
        self.task_status = {}

    def update_task_status(self, task):
        self.task_status[task.id] = task.status

class TaskQueueSystem:
    def __init__(self):
        self.task_queue = TaskQueue()
        self.task_status_tracker = TaskStatusTracker()
        self.worker_threads = []

    def add_task(self, task):
        self.task_queue.add_task(task)

    def start(self):
        for _ in range(5):  # Configurable number of worker threads
            worker_thread = WorkerThread(self.task_queue, self.task_status_tracker)
            worker_thread.start()
            self.worker_threads.append(worker_thread)

    def shutdown(self):
        for worker_thread in self.worker_threads:
            worker_thread.join()

# Example usage
if __name__ == "__main__":
    task_queue_system = TaskQueueSystem()
    task_queue_system.start()

    task1 = Task(1, 1, "Task 1")
    task2 = Task(2, 2, "Task 2")
    task3 = Task(3, 3, "Task 3")

    task_queue_system.add_task(task1)
    task_queue_system.add_task(task2)
    task_queue_system.add_task(task3)

    time.sleep(5)
    task_queue_system.shutdown()

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and provides a graceful shutdown mechanism.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and payload.
2. **Task Queue**: An in-memory data structure that stores tasks and manages their priorities.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).
5. **Retry Mechanism**: Configurable max retries for failed tasks.

### System Flow

1. **Task Addition**:
	* Client adds a task with a priority to the task queue.
	* Task is stored in the task queue with a unique ID.
2. **Task Consumption**:
	* Worker thread consumes the highest-priority task from the task queue.
	* Task is marked as "running" in the task status tracker.
3. **Task Execution**:
	* Worker thread executes the task.
	* If the task fails, it is marked as "failed" and the retry mechanism is triggered.
	* If the task succeeds, it is marked as "completed".
4. **Task Retry**:
	* If a task fails, it is retried up to the configurable max retries.
	* If all retries fail, the task is marked as "failed".
5. **Graceful Shutdown**:
	* The system is notified of an impending shutdown.
	* All worker threads are stopped, and any remaining tasks are marked as "failed".

### Overview

The in-memory task queue is designed to manage tasks with priorities, allowing worker threads to consume tasks while tracking their status. The system supports task retry with configurable max retries and provides a graceful shutdown mechanism.

### Components

1. **Task**: Represents a single task with a unique ID, priority, and payload.
2. **Task Queue**: An in-memory data structure that stores tasks and manages their priorities.
3. **Worker Thread**: Consumes tasks from the task queue, executes them, and updates their status.
4. **Task Status Tracker**: Maintains the status of each task (pending, running, completed, failed).

### System Flow

1. **Task Addition**:
	* Client adds a task with a priority to the task queue.
	* Task is stored in the task queue with a unique ID.
2. **Task Consumption**:
	* Worker thread consumes the highest-priority task from the task queue.
	* Task is marked as "running" in the task status tracker.
3. **Task Execution**:
	* Worker thread executes the task.
	* If the task fails, it is marked as "failed" and the retry mechanism is triggered.
	* If the task succeeds, it is marked as "completed".
